Troubleshooting

Summary

 If there are problems during the installation or running of CADI, the following solutions are recommended for the corresponding situations.

Error List

Problem

Reason / Action
Unable to install the ODBC drivers (database) under Windows NT One must have the correct permissions to install the driver -> ask for administrator privilege.
Authorization failed Incorrect code used -> authorize with the correct code.
Help does not start Help is not properly installed -> CADI must be reinstalled.
Data base is not found Data base is not properly installed -> CADI must be reinstalled.
Corrupted Data base/ ODBC Data Source Settings This Message will occur only during the CADI starts up. Two reasons are possible for this message.

1. The version of the CADI data base and the version of CADI are different.

Action : Reinstall CADI

 

2. The ODBC Data Source settings are invalid.

Action : Check the ODBC Data Source Settings
Functions cannot be chosen from menu, functions do not work properly or parameters cannot be changed 1. Might not be logged in -> check if logged in or not.

(In order to use some functions, it is required to be logged in)

2.  Perhaps the connection has been lost or the target has a problem -> check cables for proper connection or choose the correct COM port.

3. Might be working offline -> only a few functions can be used offline.
Logging in is a problem 1. Already logged in -> no need to log in again.

2. Incorrect password for login is used -> use correct password.

3. There is no connection -> check the cables or the COM port.
Help file has different language than that in use Presently, the Help for this language is not available -> only English Help is available at the present time.

Working Offline

 It is possible while working offline to check previously saved parameters or data.

Check the ODBC Data Source Settings
Step Description
1
  • Open "Startmenue" --> "Settings" --> "Control Panel"
  • Open the ODBC Data Sources dialog
2

Is there an entry for CADI available (possible entries: UserDSN or SystemDSN)?

  • NO: press the "Add" button and go to step 5 "Entry for CADI is not available"
  • YES: then press the "Configure" button and go to the next step.
3

Check the following adjustments.

The Database path must be correct !!

If the database path is not correct, press the "Select" button and go to the next step.
4

  • Select the correct CADI GC Directory and the CADI Database(CADI.Mdb).
  • Press the "OK" button and close all dialogs.
  • CADI GC should work now.
5

Entry for CADI is not available

Select the Microsoft Access Driver, press the "Finish" button and go to the next step.
6

Fill in the dialog, press the "Select" button and go to step 4
