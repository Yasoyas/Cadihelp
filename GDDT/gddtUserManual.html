1. Introduction

1.1 General

What is GDDT ?
Graphical Drive Diagnostic Tool (GDDT) is an HMI for Schindler VARIODYN Frequency Converters (FC), based on FCR, VarioSys and VecSys software.
The tool can be connected via RS232 interface to mains- as well as to motor control inverter.

Mains control (FCR) for PF1 FCs: Since FCR SW version V5.0 the GDDT is the exclusive conifguration tool for FCR. FCs with FCR and SW Version > 5.0 are until now: VF 155PF1AC/DC and VF288PF1AC/DC. For older PF1 FCs (VF120 / 170 / 280 PF1) with SW Version < 5.0, use the program "WinMon".

Motor control (VarioSys/VecSys): For all VARIODYN motor control FCs, the GDDT can be used as "terminal" program for 3th level diagnostics.
In addtion to a terminal program like i.e. TeraTerm, GDDT offers powerful drive specific features like a graphical tracer.
The motor control functionality is not enabled per default. For enabling it, GDDT has to be started with the attribute -terminal. See also installation manual, chapter 6 "Short cut properties".

1.2 Start GDDT

Start the program, e.g. by double clicking on GDDT shortcut.
On the next picture you can see the startup window. It can take a while until this window occurs. Please make sure, that this window is not hidden behind an other window.

The next picture shows the standard GDDT setup window.
Please note: If you start GDDT with attribute -terminal, you can also select the VarioSys drive system for motor control inverters.

Enter an appropriate session name. This name is displayed on the frame header. This helpful if you have several GDDT's sessions open, in example on a parallel inverter installation.
It's also possible to change the session name during the operation in menu "File -> Application Settings".

After you made your choice, click button "connect system".
GDDT starts after checking if the drive system is connected.

Please note, that a drive system must be connected via the RS232 interface.
It's not possbile to start GDDT without a drive system.

2. Use of GDDT with FCR

2.1 Drive Configure

2.1.1 Parameter

2.1.1.1. Standard parameterisation procedure

  1. Open the parameter dialog: ->Drive Configure->Parameter
  2. Each time the parameter window has been opened, GDDT uploads the current parameters from FCR and sets the uploaded values to column "Upload". Please note that the menus and buttons are blocked until all parameters are uploaded.
  3. The first time you open the parameter window, the FCR system selection window starts up automatically.
  4. In the FCR system selection window, select your drive system, mains and motor voltage.
  5. Press the button "set to work/download col. and download".
  6. GDDT starts the download procedure
    1. All parameters are calculated and put in the "Work/Download" column.
    2. All selected parameters will be downloaded to FCR.
    3. After downloading, GDDT invokes the save to memory procedure in FCR.
    4. The paramteres are now stored persistently on FCR.
    5. GDDT again uploads the parameters and sets the uploaded values to column "Upload".
    6. The uploaded parameters are checked against the downloaded parameters.
  7. After successful download, persistent storage, upload and verification GDDT displays the action status: "Download successful !"

2.1.1.2. Meaning of the buttons

Button "define"
Opens the FCR system selection window.

Button "upload"
Uploads the current parameters from FCR and sets the uploaded values to column "Upload".

Button "download"

  1. Downloads the parameters from work/download column to FCR.
  2. Stores the values on FCR.
  3. Uploads the parameters and sets the uploaded values to column "Upload".
  4. Compares the values in the "Work/Download" column with the values inside the "Upload" column.

2.1.1.3. Various

Please note: Try again if a procedure failes. This can happen due to interferences on RS232.

2.1.2 Set Time and Date

Standard procedure

  1. Open the "Set Time and Date" dialog: ->Drive Configure->Time and Date
  2. Each time the "Set Time Date" window has been opened, GDDT uploads the current time and date from FCR and sets the values inside the edit table.
  3. If the loaded values are not correct, you can set new values either by pressing the button "Load PC time" or by pressing the button "Edit manually" and setting the values manually inside the edit table.
  4. Press the button "Download edit table values".
  5. GDDT sets the time and date on FCR according the values inside the edit table.
  6. Press the button "Load FCR time" and verify the correct function of the FCR clock.

2.2.4 Tracer

The tracer offers you a powerful tool to visualise FCR control records.

Enter the trace mode by reseting the FCR. Please note that after a reset the tracer is loaded with random values. The tracer mode will be left as soon as an other command e.g. read out errorlog is executed. The next picture shows the dialog for entering in trace mode

FCR Tracer
The FCR tracer samples 500 values with a step of 1ms. The tracer restarts at each trip start (PWM enable) as well as when the button "Trigger" is pressed.

"Trace" button
With the "Trace" button you read out the trace buffer. The values are displayed in the graphical window.

"Trigger" button
The trace buffer is restarted. The "Trigger" button can be pressed during a trip.

The next picture shows a trace in the graphical window.

Signal selection
You can hide the curves of no interest by removing the tick in the check box.

Zoom function

  1. GDDT offers a powerful zoom function. Click an drag with the mouse the part of the graphic you would like to enlarge.

  2. After releasing the mouse button, you'll see the zoomed section.

  3. To get back to the orginal curves press the "Refresh" button.

Cursor
Two vertical cursors are available. The signal values on cursor position are displayed in the table.
Cx Rough value = the rough value from FCR signal on cursor position.
Cx Value = normed value on cursor position.

Norm
The signal value is multiplied with the value in column "Norm" to get the absolute signal value. The "Norm" value is adjustable.

3. Use of GDDT with Variosys / Vecsys

Motor control (VarioSys/VecSys): For all VARIODYN motor control FCs, the GDDT can be used as "terminal" program for 3th level diagnostics.
In addition to a terminal program like i.e. TeraTerm, GDDT offers powerful drive specific features like a graphical tracer.
The motor control functionality is not enabled per default. For enabling it, GDDT has to be started with the attribute -terminal. For more information see installation manual, chapter 6.

3.1 Terminal

The next picture shows the terminal menu. Some helpful commands for terminal handling can be found.

Logging terminal output to a file

  • Browse Log to File
    A browser will be opened and a file name can be entered manually
  • Auto Log to File
    GDDT generates automatically a file name depending on drive system, session name, date and time
  • Stop Log to File
    Logging to file is stopped
  • Pause / Resume File Logging
    Logging to file is paused / resumed
* = This buttons are only visible when the logger is active

Insert Comments
After pressing the menu item "Insert Comment", it is possible to write comments in the output flow window. RS232 communication is interrupted until "Comment done"* is pressed.
* = This menu item is only visible when comment mode is active.

Command View
A window with drive commands gets visible. Drive commands can be selected. With the button "WRITE" the command is sent to the FC.
Be careful by using this commands, wrong use can have serious impact on drive functionality !

3.2 Tracer

The tracer offers you a powerful tool to visualise motor control records.

Motor control tracer on Variosys/Vecsys
The tracer restarts at each trip start. Normally the sample time is 10 ms. This period and the measurands can be adjusted with terminal command "Z". The trace buffer size is limited. On long trips, the start sequence may be lost.

Trace button
With the trace button you read out the trace buffer. The values are displayed in the graphical window.
