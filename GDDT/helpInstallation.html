1. Introduction

This document describes how to install Graphical Drive Diagnostic Tool (GDDT).
GDDT is a JAVA program. GDDT requires a JAVA virtual machine, also called Java Runtime Environment (JRE) installed on your PC. In addition, GDDT requires some extended library files installed inside the JRE. If you install GDDT according chapter 2 "automatic installation via CADI" , the JRE of CADI will be used and all files will be installed automatically.

2. Automatic installation via CADI

This chapter describes the automatic GDDT installation. The whole file structure (see chapter 5) will be generated automatically.

  • Install CADI GC version 2.92 or later on your PC.
  • The installation of CADI contains the installation of GDDT.

The next picture shows the CADI program group, containing GDDT.

The next picture shows the GDDT shortcut.

In the shortcut properties (chapter 6), you can choose between standard functionality (FCR only) or an additional terminal functionality, which contains a powerful graphical trace terminal for motor control inverters (VarioSys/VecSys).

3. Update to the latest GDDT version

You have already installed CADI GC with GDDT, but you need to update to the latest GDDT version.

  • Replace the old GDDT.jar file with the new file.
  • The GDDT.jar file is located in the CADI top level folder: E.g. C:\Program Files\CADI_GC (see also chapter 5 "file structure").
  • If requested, replace also other files inside the GDDT file structure (see also chapter 5 "file structure").

4. Manual installation with or without CADI

  • Either the CADI GC version >= 2.90 must be installed on your PC or an JAVA JRE >= version 1.5.
  • Copy all files according chapter (5) "file structure":
    • Application files: Either in the CADI top level folder or in any directory you want.
    • JRE files: Either in the CADI JRE or the JRE you would like to use.
    • Adapt the JRE path to your JRE. See also chapter 6 "short cut properties"..

5. File Structure

This chapter provides a description of the GDDT specific file structure.
If you install GDDT according chapter 2 "automatic installation via CADI" , this files will be installed automatically.

Application specific files

Root directory: e.g. C:\Program Files\CADI_GC

  • GDDT.jar -> This is the main application file
  • GDDT shortcut -> Shortcut, containing startup attributes
  • GDDT.ico -> Program icon
  • Folder Help
    • Folder gddt
      • index.html -> help file index
      • helpInstallation.html -> help file installation
      • more html files...
      • Folder images
        • various images...
      • Folder gddt_default_files
        • FCRDefaultParameter.xml -> Files containing FCR parameters
        • DriveSettingsFCR.xml -> Files containing FCR settings
        • ApplicationSettings.xml -> User settings are stored in this file
    • Folder fcr
      • Parameter_50.html -> Fcr parameter description
      • Error_50.html -> Fcr error description
Additional JRE specific files

Root directory: e.g. C:\Program Files\CADI_GC\jre

  • Folder bin
    • win32com.dll -> dll for com port*
  • Folder lib
    • Folder ext
      • plot.jar -> library for graphical display
      • comm.jar -> library for serial port

* = When automatic CADI installation, this file is placed in the CADI root directory e.g. C:\Program Files\CADI_GC\

6. Shortcut properties

6.1 Adapt the path to your virtual machine

  • The shortcut starts (property "Start in:") where the application file GDDT.jar is located
    E.g.: D:\CADI_GC\
  • In property "Target", verify the path to the java virual machine (javaw.exe)
    The javaw.exe file is normally located in the CADI jre\bin folder.
    Target command line e.g.: D:\CADI_GC\jre\bin\javaw.exe -jar GDDT.jar

6.2 GDDT used as terminal with graphical tracer for Variosys

  • GDDT can be used as terminal for Variosys/Vecsys motor control inverters. This terminal contains a powerful graphical tracer.
    For enabling this function, you have to add the attribute -terminal at the end of the target call.
  • The picture below shows the modified target call inside GDDT shortcut properties.
    Add -terminal as attribute behind the GDDT.jar call:
    e.g.: D:\CADI_GC\jre\bin\javaw.exe -jar GDDT.jar -terminal
