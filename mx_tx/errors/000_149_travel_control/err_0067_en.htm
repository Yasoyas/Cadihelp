0067 KUET_Fault

Summary

 This section describes the Travel Control error KUET_Fault.

Description

The monitoring of the door bridging signals (feedback from relays RUET and RUET1) indicates that they are not working properly. The relays RUET and RUET1 are not switching in parallel, or RUET and RUET1 are switching outside the door zone fin/magnet.

Comments: Both signals are transmitted via bus communication from the car. The error does not trigger an emergency stop. An error detected during a trip is logged only after the car has come to rest. Therefore the trip-phase stored in the log always appears as STANDSTILL.

This error is also generated on elevators with MOSIS system. For detailed information about the error cause please refer to the description of the extra information below. With MoSIS this error is also logged when there there is a mismatch between the numbers of floors learnt by MOSIS and by the control system.

Extra Info

The extra information value is a 8 digit hexadecimal number (0xABCDEFGH) and has to be converted/decoded into binary format (each bit has a specific meaning).

Meaning of value 'GH' (Bit0 to Bit7):
0000 0001(bin) / 01(hex): Status of KUET and KUET1 are different
0000 0010(bin) / 02(hex): Enable UET false or outside a floor but KUET or KUET1 is TRUE
0000 0011(bin) / 03(hex): Enable UET true and at floor but KUET or KUET1 is FALSE
0000 0100(bin) / 04(hex): Status of KUET_20 and KUET1_20 are different
0000 0101(bin) / 05(hex): Enable UET_20 false or outside a floor but KUET_20 or KUET1_20 is TRUE
Additional diagnostics for MX only:
0000 0110(bin) / 06(hex): Car at floor without door on side1 but IUET or IUET1 is true
0000 0111(bin) / 07(hex): Car at floor with door on side1 but IUET or IUET1 is false
0000 1000(bin) / 08(hex): Car outside floor but IUET or IUET1 is true
0000 1001(bin) / 09(hex): Car at floor without door on side20 but IUET_20 or IUET1_20 is true
0000 1010(bin) / 0A(hex): Car at floor with door on side20 but IUET_20 or IUET1_20 is false
0000 1011(bin) / 0B(hex): Car outside floor but IUET_20 or IUET1_20 is true

Meaning of value 'F' (Bit8 to Bit11):
0000(bin) / 00(hex): 'KUET' false, 'KUET1' false, 'Enable UET' false
0001(bin) / 01(hex): 'KUET' true, 'KUET1' false, 'Enable UET' false
0010(bin) / 02(hex): 'KUET' false, 'KUET1' true, 'Enable UET' false
0011(bin) / 03(hex): 'KUET' true, 'KUET1' true, 'Enable UET' false
0100(bin) / 04(hex): 'KUET' false, 'KUET1' false, 'Enable UET' true
0101(bin) / 05(hex): 'KUET' true, 'KUET1' false, 'Enable UET' true
0110(bin) / 06(hex): 'KUET' false, 'KUET1' true, 'Enable UET' true
0111(bin) / 07(hex): 'KUET' true, 'KUET1' true, 'Enable UET' true

Meaning of value 'E' (Bit12 to Bit15): for TX only
0000(bin) / 00(hex): 'PHS' = OUT_FIN, 'PHNR_D' = OUT_FIN
0001(bin) / 01(hex): 'PHS' = IN_FIN, 'PHNR_D' = OUT_FIN
0010(bin) / 02(hex): 'PHS' = OUT_FIN, 'PHNR_D' = IN_FIN
0011(bin) / 03(hex): 'PHS' = IN_FIN, 'PHNR_D' = IN_FIN

Meaning of value 'D' (Bit16 to Bit19): for MX only
0000(bin) / 00(hex): 'IUET' false, 'IUET1' false, 'LUET' false
0001(bin) / 01(hex): 'IUET' true, 'IUET1' false, 'LUET' false
0010(bin) / 02(hex): 'IUET' false, 'IUET1' true, 'LUET' false
0011(bin) / 03(hex): 'IUET' true, 'IUET1' true, 'LUET' false
0100(bin) / 04(hex): 'IUET' false, 'IUET1' false, 'LUET' true

0110(bin) / 06(hex): 'IUET' false, 'IUET1' true, 'LUET' true
0111(bin) / 07(hex): 'IUET' true, 'IUET1' true, 'LUET' true

Meaning of value ‘AB’ (Bit24 to Bit31): for MoSIS only (bit decoding)
0000 0001(bin) / 01(hex): MoUET_RelaisFail is ON.
0000 0010(bin) / 02(hex): MoUET_WrongFloors is ON.
0000 0100(bin) / 04(hex): MoUET_reserve2 is ON.

0100 0000(bin) / 40(hex): MoUET_v_below is ON.
1000 0000(bin) / 80(hex): MoUET_OK is ON.
Meaning of Value “CDEFGH” (Bit0 to Bit23): none (always '000000')
Example 1: Error 67 with extra info value h68000000 on a MoSIS system
68 (Hex) / 01101000(bin): MoUET_inLuet and MoUET_inDoorZone and MoUET_v_below are ON.
Example 2: Error 67 with extra info value h31000000 on a MOSIS system:
31(Hex) / 00110001(bin): MoUET_RelaisFail and MoUET_CmdOn and MoUET_inDoorZone are ON.

Causes and Actions
ID Possible causes of message Actions to be taken
1 Bad wiring on car Check the connection of the RUET/RUET1 enable- and feedback-signals to the car interface board.
2 Check the wiring of the door zone detection devices (PHUET / KUET)
3 Bad environmental conditions (dirt, dust) in the hoistway Check photocell/magnet switches on car for dirt.
4 Electrical problems Check the ground connection on the car and the power supply for the car interface board and the door bridging device (e.g. PHUET)
5 Communication problems between control cabinet and car Check the (LON) bus connection from the control cabinet to the car. Caution: KUET error may indicate a communication problem caused by another LON node (landings etc.)
6 Check for EMC problems caused by Schindler or 3rd party equipment (MX-GC: refer to field information No.291)
7 Configuration Check if number of floors and shaft layout (front rear doors) in configuration is matching the floor vanes in the hoistway.
MX-GC: Travel control checks for each access is a KUET vane is present for front and/or rear side.
8 Mechanical problem with switches Check mounting of KUET switches/fins
9 With MoSIS: MOSIS was replaced, e.g. from other installation (learnt floors mismatch) Perform a learn trip.
