0089 TripTimeExp

Summary

 This section describes the Travel Control error TripTimeExp.

Description

The maximum time for the duration of a trip was exceeded. The monitoring of the trip time is required as per EN81-70 (chapt. 12.10) and aims to prevent damage (mechanical wear) of the moving parts (traction sheave, ropes) and damage of the motor, the brake and the inverter caused by overheating. For SW releases dated earlier than April 2006 the maximum trip time is defined by the parameter 'MAX_TRIP_TIME' only (see also description of extra info). For software releases with later date the software calculates the maximum trip time before each trip based on the maximum speed for the selected trip type and the maximum possible travel distance (HQ) and uses the calculated value to monitor the trip (if the maximum travel distance is not known the parameter 'MAX_TRIP_TIME' is used instead).

Extra Info

0: The maximum time value for the monitoring is defined by the parameter 'MAX_TRIP_TIME'.

With extra info value not 0: aabbccdd(hex) -> 8 digit hexadecimal number
aa: Calculated trip time in 5 sec resolution (e.g. 0A = 50seconds)
bb: Start floor (floor number) of the trip for which the trip time is (re-) calculated.
cc: Target floor (floor number) of the trip
dd: bit-encoded value for diagnostics (bit8 MSB to bit1 LSB).
(dd)Bit8 = 1: Trip velocity zero
(dd)Bit7-Bit1: Trip type identification

Source ID

None.

Causes and Actions
ID Possible causes of message Actions to be taken
