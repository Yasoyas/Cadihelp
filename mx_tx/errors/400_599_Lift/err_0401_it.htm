0401 VCOM Node Connected

Summary

 This section describes the Lift error VCOM Node Connected.

Description

The control software detects a new VCOM bus node (subsystem) on the bus. The message is logged when the connection to the node/subsystem is established after it was interrupted for some time or if the node/subsystem is reinitialising. A manual reset of the elevator control causes this message to be logged on the other elevators in the group.

Extra Info

Identification number of the VCOM node (8 digit hex. value xABCDEFGH).

Value of digits ABCDEF: information for R&D only
Value of digits GH: identification number of the VCOM node (pcb)

Values for VCOM node identification number:
Old encoding "VXT_SUBSYS_ID":
x07 = Controller Lift 1
x08 = Controller Lift 2
x09 = Controller Lift 3
x0A = Controller Lift 4
x0B = Controller Lift 5
x0C = Controller Lift 6
x0D = Controller Lift 7
x0E = Controller Lift 8

New encoding "LOGADDR" (logical address of the node):
x11 = Controller Lift 1
x21 = Controller Lift 2
x31 = Controller Lift 3
x41 = Controller Lift 4
x51 = Controller Lift 5
x61 = Controller Lift 6
x71 = Controller Lift 7
x81 = Controller Lift 8
xD9 = E-Vision PC
xFA = SID Workstatoin (PC #1)
xF9 = SID Workstation (PC #2)
xF8 = Gateway Interface Board (GW)
xD1 = SID embedded Lift 1 (MBRD Print)
xD2 = SID embedded Lift 2 (MBRD Print)
xD3 = SID embedded Lift 3 (MBRD Print)
xD4 = SID embedded Lift 4 (MBRD Print)
xD5 = SID embedded Lift 5 (MBRD Print)
xD6 = SID embedded Lift 6 (MBRD Print)
xD7 = SID embedded Lift 7 (MBRD Print)
xD8 = SID embedded Lift 8 (MBRD Print)

Example:
Extra info value = 0000E131: Controller of Elevator #3 connected to VCOM
Extra info value = 0000E5FA: SID PC (#1) connected to VCOM

The values and the encoding of the extra info for TX4 (RS/IVXVF/SIOL) systems are not described here.

Source ID

None.

Causes and Actions
ID Possible causes of message Actions to be taken
1 hardware fault, hardware reset If this message appears very often without an obvious reason, check the VCOM bus connection and test the corresponding VCOM node to verify its operation; exchange it if necessary.
