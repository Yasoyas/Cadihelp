1219 TmoDatGP_Konf

Summary

 This section describes the LON error TmoDatGP_Konf.

Description

The SIO requests PCT configuration checksum information from the RS after every reset. If the checksum is different, hte configuration data is downloaded. This error is generated by the SIO when the checksum for this paramater group was received, but the data was not received and a timeout occurrs.
On the SIOG, the PCT information is requested from the lowest numbered lift first. If the information is not received, the SIOG continues to try each lift in turn until the information is received.
The SIOL always downloads its PCT configuration from the RS in the same lift.

Extra Info

SIOL: not used
SIOG: lift-ID of the RS which did not respond

Source ID

SIOL or SIOG

Causes and Actions
ID Possible causes of message Actions to be taken
1 Incorrect or corrupted PCT Download a correct configuration file.
2 Error on RS Save error logs and call EBI/RAND
