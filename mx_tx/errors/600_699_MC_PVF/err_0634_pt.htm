0634 Exception vector

Summary

 This section describes the MC PVF error Exception vector.

Description

The PVF processor detects an internal fault (e.g. division by zero). After this error is generated, the CPU peripheral is reset (means reset on PVF board). The Id, type ot the exception, date, time, trip number, etc. and the contents of all CPU registers are saved in the CPU Status Log. Date, time, trip number, etc. are saved as usual in the Error Log too. For diagnostics, the PVF "CPU Status Log" & "CPU Staus Counters" should be used as well as "Warning & Error Logs" and "Warning & Error Counters".
Normally this error should not occur. If it is not possible to remove the problem using the actions described below, contact Ebikon, dept. R&D-DR.

Extra Info

None.

Source ID

None.

Causes and Actions
ID Possible causes of message Actions to be taken
1 Absolutely wrong parameters Check PVF parameters.
2 Check PVF CPU status log: If the event zero divide is stored, some parameters must be wrong.
3 RAM not properly deleted after a change ot the software. Remove the two RAM (IC 76, 77) at least for 10 sec. This is absolutely necessary, if the PVF SW (EPROMs) has been changed.
4 CPU or peripherals failure. Disturbance or interference. Change PVF board.
5 Bad power supply (5V) Check UVVF or UKE2.
