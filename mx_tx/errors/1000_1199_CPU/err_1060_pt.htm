1060 BErr MoveMultiple

Summary

 This section describes the CPU error BErr MoveMultiple.

Description

The CPU was reset due to a Bus Error while reading or writing multiple values to a memory location that does not exist, is restricted to read-only access, or is restricted to supervisor-only access. Move-Multiple usually happens when the CPU saves or restores several register values simultaneously at the beginning or end of a function call. This may be due to the User Stack Pointer being corrupted.
This can happen with instructions like:
MOVEM A5/A4/A3/A2/A1/D3/D2/D1/D0, -(A7)

Extra Info

Program counter before the fault

Source ID

None.
