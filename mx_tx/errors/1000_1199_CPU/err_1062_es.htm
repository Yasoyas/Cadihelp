1062 Dead Task

Summary

 This section describes the CPU error Dead Task.

Description

The CPU was reset due to a thread that didn't report itself alive within the corresponding supervision interval. Typically, the system is still running, but a lower priority thread does not get enough CPU time because a high priority task uses too much of the CPU resource.

Note, this is NOT the same as a hardware watchdog reset. See error 1052 for a description of the hardware watchdog reset.

Extra Info

Description of which tasks didn't respond, varies according to SW version. (For R&D internal use)

Source ID

None.
