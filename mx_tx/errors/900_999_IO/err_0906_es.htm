0906 Duplicate_BMK

Summary

 This section describes the IO error Duplicate_BMK.

Description

If the same critical BMK is configured more than once on the same lift then the error will be logged. In case the software finds a unallowed duplicate entry in the IO database (e.g. for a critical input) an error will be logged for each entry. The first number in the extra info indicates the sequence of the entries in the IO database (the same BMK may be configured twice on the same node).

Extra Info

Identifies the board type of the bus node on which the same BMKs are configured. xABCDEFGH (8 digit hex number) where:

A: is the sequence number of the entry in the internal IO database
B: Identifies the IO Domain
CD: Node type
EF: Higher byte of Node ID
GH: Lower byte of Node ID

Identification numbers of the sequence number (A):
1: means the error is for first found node entry in the IO database
2: means the error is for the other node

Identification numbers of the IO Domains (B):
1 for IO_DOMAIN_LOCAL
2 for IO_DOMAIN_LON
3 for IO_DOMAIN_BIO2
4 for IO_DOMAIN_CAN (Not in used yet)

For explanation of the LON node types please refer to the description of error 1209.
