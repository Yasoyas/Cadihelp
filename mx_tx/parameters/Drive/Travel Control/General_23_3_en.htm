General

Summary

 This section describes the parameters in the parameter window General of the parameter group Travel Control in subsystem Drive.

Allow Inspection without JRECC

This parameter is only changed for installations in North America.

 














Upper access floor


The control buttons for hoistway access control are installed on two floors. 
The access to the car top is possible from the floor defined with P_ANT_UPPER ACCESS FLOOR (normally at the top end/floor of the hoistway).
The access to the pit is possible from the bottom floor level (lowest floor). The elevator can only be moved using the control buttons (up/dow) if it is at either of the two floors.





















Max Trip Time


For SW releases dated earlier than April 2006 the trip time is limited by the parameter 'MAX_TRIP_TIME' only (see also description of error 89). For software releases with later date the software calculates the maximum trip time before each trip based on the maximum speed for the selected trip type and the maximum possible travel distance (HQ) and uses the calculated value to monitor the trip. If the maximum travel distance is not known the parameter 'MAX_TRIP_TIME' is used instead (learn trip). If the time elapses during a trip the trip is interrupted and the error message 'TRIP_TIME_EXCEEDED' is logged.

Default value for TX: 700s
Default value for MX: 700s
refer to EN81-1:1998 chapter 12.10

Parameter description for software released before April 2006:
Time limit for the monitoring of the trip time during automatic trips. If this time elapses during a trip the error message 89 TRIP_TIME_EXCEEDED will be generated.





















Drive Type


Type of the electrical drive. The parameter value is determined automatically once the serial communication to the drive is established.

0: ACVF_EU_AP
1: ACVF_NA
2: SSDC
3: WLDC
4: VECSYS_EU_AP
5: VECSYS_NA
6: MAGNETEK
7: H_BRIDGE
8: YASKAWA
9: CDI
10: Invalid Drive Type

Type 'CDI' added 17.10.2005





















Safety circuit type


Defines the type of safety chain in use:

values:
0: TX_OLD_SAFETY                 IVXVF-TX with old safety chain
1: TX_OLD_SAFETY_1SH         IVXVF-TX old safety chain '1 Contactor'
2: TX__SAFETY                       IVXVF-TX with new safety chain
3: VDS_PGO                           VDS-New safety chain
4: TX_S700_SKEAP                 GCIO-S700 with SKEAP
5: TX_S700_SKENA                 GCIO-S700 with SKENA
6: MX_SAFETY                        GCIO-MX with new safety chain
