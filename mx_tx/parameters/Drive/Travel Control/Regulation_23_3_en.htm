Regulation

Summary

 This section describes the parameters in the parameter window Regulation of the parameter group Travel Control in subsystem Drive.

P-Ampli Accel

Amplification for the proportional section of the position controller during acceleration and at constant speed.
Values greater than 20 can reduce passenger comfort.

 














P-Ampli Decel


Amplification for the proportional section of the position controller during deceleration. With a value of 10 the position control is smoother and comfort is enhanced, but the car will stop less precisely. As a general rule the value for P_ANT_P_AMPLI_DECE should be kept at a low value and leveling accuracy should be optimized using parameters P_ANT_I_AMPLI_DECE and P_ANT_P_AMPLI_KEF.
Values greater than 50 can reduce passenger comfort.





















KEF Enable


The parameter P-Ampli KEF is activated in the KEF area. In addition, when the photo-electric cell enters the floor vane the measured distance for leveling-in correction (KEFTOT) is not transmitted directly to the position controller (direct addition to DELTA_S), but via a filter which adds an increment to DELTA_S every 10 ms until the whole KEFTOT value has been transmitted.





















P-Ampli KEF


Additional P (proportional) part which is added to the existing P-part 185 mm before the floor (range of Korrektur EinFahrt), when the first photo-electric cell enters the vane. 
The value of P_ANT_P_AMPLI_KEF must be approximately the same as P_ANT_P_AMPLI_DECE so that an effect is visible. The value of P_ANT_P_AMPLI_KEF can also be negative in order to reduce the whole P-part in this area.





















Pre Indice


The distance regulation starts its output of velocity_reference values the time earlier as defined in Pre Indice.





















I Begin


Defines the start of the I (integral) section during the deceleration phase.
This time before destination is added to the I section. This activates parameter P_ANT_I_AMPLI_DECE. For high values the I section is active during the whole deceleration phase but not before.





















I Ampli Decel


Integral portion for the amplification of the position controller. The position control is active during the final landing approach of the car (last few ms of deceleration phase). This parameter influences the stopping behaviour/characteristics. Switch-on time for the I section is set with the parameter P_ANT_I_BEGIN.

Caution: Very high parameter values destabilize the system. The elevator turns away from the destination floor and the events 'FALSE DIR' and 'ERR RELEVEL' occur during the fourth rounding of the speed curve (turning point).





















Time Stop Precision


See comment (Note) on the parameter Dist Stop Precision.





















Time Advanced Selector


The Advanced Selector is the selector message to the control which is generated before the actual selector. This allows the system to accommodate conflicting destination and selector messages. This parameter is intended only for development work. If this parameter is changed it can result in destination overrun.





















Window Auto Tacho Factor


During constant-speed IVXVF may recalculate his tacho-factor according the values of position- & speedtacho.
To deactivate the automatic tachocorrection set this parameter to 0.
Default value: 50 [Tachopulses/sec.]





















Result Auto Tacho Factor


There is no action if you write any value to this parameter. The program will overwrite this parameter each time if it recalculates the tachfactor.





















Acceleration Tolerance


Acceleration limitation is activated after the position controller to accommodate any jumps in actual values (e.g. when tacho slip is corrected). This ensures that the acceleration calculated from the speed reference values generated remains within specific tolerance limits about the acceleration reference value.
At values above 300 the tolerance is so wide that limitation is no longer effective.





















Rope Elongation


Defines the (dynamic) rope elongation during the deceleration of the car. On high rise elevators extra rope stretch on the hoisting ropes may be caused during the acceleration and deceleration phases. The parameter is used by the software to compensate the measured car position in this situation. The rope elongation compensation takes this dynamic factor into account and corrects the 'S_IST' (current car position) for the position controller accordingly. The calculated compensation value is proportional to the current acceleration/deceleration of the car and the value of the parameter 'Rope Elongation'.
The compensation is usually only needed on high rise buildings (>100 meter) and for control systems where the car position is measured with an encoder mounted on the overspeed governor. Therefore the compensation is not required with Mosis.

See also parameter 'P_ANT_GOVERNOR_ROPE_ELONG'





















Weight House Shrink


In normal operation the control compensates continuously the shrinkage/expansion of a building due to environmental conditions (temperature) or ageing by updating the floor positions (shaft image) learnt during the learn trip (the learnt values can be made visible with CADI). Every time the car stops on a floor it measures the position difference between the value stored for this floor by the control and the current value from the car position information. If a deviation is detected a separate correction value per floor is updated accordingly. After 10 arrivals of the car on a floor the average correction value for that floor is added/subtracted to/from the current position value. Before adding/subtracting this correction value the value itself is scaled (multiplied) with the parameter 'weight_house_shrink' in order to allow that only a proportional part of the pre-calculated correction value is finally added to the floor position values.
With a value of 0 the correction is disabled.





















Distance Stop Precision


This is activated after the predetermined time for the trip has elapsed. The position controller remains active until this leveling accuracy has been achieved or until the Time Stop Precision parameter time has elapsed.
The I section of the position controller will increase during this phase in order to position the elevator at the floor within the leveling accuracy values defined in Dist Stop Precision.
If the value is set at 1 the process takes proportionately longer. In such a case the parameter Time Stop Precision must be matched accordingly.





















Door Contact Delay


Delay during trip initiation to avoid emergency stops caused by bouncing door contacts. Defines the time delay from the moment the door(s) has closed until travel control disables the door contact bridgeing in the safety circuit.

If the door contacts bounce (reopen) within this time limit the preparation for a new trip is interrupted and the event SAFETY_RTS or SAFETY_END is generated.

This parameter is effectiv from TXGC 1.46 and higher. It replaces the Parameter (P_ANT_RESERVE_SHORT_2)





















Releveling Delay


Drive Relevelling Delay





















Governor Rope Elongation


Defines the elongation of the governor rope during deceleration of the car (at 1 m/s*s) at the bottom floor in tacho units [TF]. This parameter is used by the software to compensate the measured car position when the car enters into the fin of the targetfloor. It is usually only needed only for high rise buildings (>100 meter) and for control systems where the car position is measured with an encoder mounted on the overspeed governor. The compensation is not required with Mosis. The typical effect observed when the compensation is not set as it should be is that the car stops with different distances away from the floor level after trips from below and above the floor. A value of 0 means no compensation of the car position is done by the software (default).

Adjustment: If the stopping position on the second floor is different in case of approaching from top and from bottom (e.g. +5mm approaching in up direction and -2mm approaching in down direction), the value should be the difference of the deviation times 2 in tacho units (e.g. (5 - (-2))/2 = 3.5mm. If tacho factor is 3400 the value is 12).

See also parameter 'P_ANT_ROPE_ELONGATION'





















Speed Regulation Mode


The parameter allows to select the regulation mode. 

0: Speed regulation with (conventional) time based calculation of speed reference curve.
1: Speed regulation with distance based calculation of car reference speed.

On new installations the conventional regulation is used normally. For modernization (existing motors) the distance based regulation can be activated in order to avoid 'underspeed' errors due to slow running of the motor in certain operating conditions.
