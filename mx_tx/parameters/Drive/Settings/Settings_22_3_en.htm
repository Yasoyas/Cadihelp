Summary

 This section describes the parameters in the parameter window Settings of the parameter group Settings in subsystem Drive.

Id Motor


Identification of the motor. The motor parameters are derived from the parameter table on the inverter based on the value for the motor setting. This parameter needs to be edited only if the motor parameters selected are not commissioned individually.

Value: 4 last digits of the Schindler Id.No.
Example: the motor VM132-C4A240 is identified with the Id.No. 147964. Therefore, the correct value for the Motor Id is 7964.

The value 0 corresponds to NO_SETTING. A negative value indicates which value the parameter had originally i.e. before at least one motor parameter was edited and will be set automatically by software.. 
All motor parameters are defined in DR-92069_Vxx.

Gear Type


Defines the type of the gear.

{0.0 = NO_GEAR}	[COS]	direct traction drives (gearless)

{1.0 = WORM_GEAR}	[COS]	all worm gears (for instance Schindler worm gears: W140, W163, W200, W250, W53, W54, W55, W56, W57, W58, W59)

{2.0 = PLANET_GEAR}	[COS]	all planetary gears (high efficient gears, e.g. P420, P450)

{3.0 = SGB}	[COS]	"Semi Gearless Belt" drives for modernization

Parameter also sets the control parameters: P Gain Speed, Zero Speed Amp, Speed Edge. The value WORM_GEAR activates a control algorithm that compensates the static friction of worm gears. The values NO_GEAR, PLANET_GEAR and SGB activate the torque feed forward algorithm that calculates the required torque depending on the actual acceleration and Mass Moment of Inertia. The values WORM_GEAR, PLANET_GEAR and SGB activate the fast speed control at the beginning of the ride, if there is no DMS load measurement.

The parameter P_ANT_BRAKE_TYPE is derived from value of P_ANT_GEAR_TYPE (update function).





















Encoder Type


Defines the type of the encoder on the motor.

{0 = INCREMENTAL}	all inverters with optical, incremental encoder 
(IG 500, IG1024, IG 2000, IG2048 or IG10000)

{1 = SINE_WAVE}	optical, sinusoidal encoder. (IG1024 for planetary gear FMP160, IG5000 for FM560 / FM630 / FM710)

{2 = NO_ENCODER}	inverters without incremental encoder. Only used for modernisations. Causes the inverter to activate the slip control instead of the vector control.

{3 = SINCOS_ENDAT}  optical, sinusoidal encoder with ENDAT-interface.

{4 = SINCOS_HYPERFACE}  optical, sinusoidal encoder with Hyperface-interface.

Exception: If at the same time a planetary gear and an encoder with 1024 periods per revolution is used, set to {1 = SINE_WAVE}.

Nominal Load


Nominal car load [kg]. For double deck elevators this is the sum of the nominal loads for the lower and the upper deck. E.g. if a DD elevator is specified with a nominal load of 2 x 1600kg then the value for P_ANT_NOM_LOAD is 3200kg.

On first generation VX/TX systems, nom_load was defined twice (also in the LIFT subgroup of the PCT). For TX-GC2/MX only the P_ANT_NOM_LOAD is required.

The calculation of the drive internal load limit values for the load-measuring system is based on the parameters NOM_LOAD and CODE_TYPE
for NA:
kl_h_load = NOM_LOAD / 2 ,
kl_max_load = NOM_LOAD * 1.1 ,
kl_x_load = NOM_LOAD * 1.3 ,

others (EUAP):
kl_h_load = NOM_LOAD / 2 ,
kl_x_load = NOM_LOAD * 1.1 ,

these limits are independent of load limits defined in PCT_GP_RS_GENERAL

Phase Direction


Sense of rotation of motor (in case of induction motors the parameter is only used for machines without incremental encoder)

Parameter extended from software versions: Variosys V1.03 and Vecsys V2.15
'Phase_Dir' determines the direction (sense) of rotation of the motor and the sense of rotation of the encoder. The value depends on the required direction of rotation of the motor, the orientation of the motor with regard to the ropes and the car and the orientation of the encoder.

Induction motors (designation: VM, FM, AM, ....):
The parameter value is defined by the commissionig step "Direction Test" ("T3" respectively). It depends on the direction of rotation of the motor, the orientation of the motor with regard to the ropes and the car and the orientation of the encoder.
Possible values: 0, 1, 2, 3.

Synchronous motors (designation: PMS, SM, ....):
The value has to be set according to the following rules.
PMS230/PMS300/PMA machine:
{0}:	The motor rotates counterclockwise (seen from the traction sheave), if the elevator travels upwards.
{1}:	The motor rotates clockwise (seen from the traction sheave), if the elevator travels upwards.
PMS230/PMS300/PMA machines will not run with values 2 or 3 unless motor phases are not wired correctly.

PMS420/SM (Zetatop) machine (motors that correspond to the standard EN 60034-8:2002):
{2}:	The motor rotates counterclockwise (seen from the traction sheave), if the elevator travels upwards.
{3}	The motor rotates clockwise (seen from the traction sheave),  if the elevator travels upwards.
PMS420/SM (Zetatop) machines will not run with values 2 or 3 unless motor phases are not wired correctly.
