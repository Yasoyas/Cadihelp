Card Reader

Summary

 This section describes the parameters in the parameter window Card Reader of the parameter group Services in subsystem Lift Control.

Card Reader Mode

Card reader mode used for access control functions ZBC3 and ZBD1 (see ESS)

0: Card reader function disabled (elevator goes out of service JAB when car reader operation is active)

1: Card reader function enabled. Activation of a KKL signal temporarily enables car calls locked by HW inputs.

2: Card reader function enabled. Activation of a KKL signal temporarily enables car calls locked by ICC Lobby.

3: Card reader function enabled. Activation of a KKL signal temporarily enables car calls locked from any source (ICC Lobby or HW inputs).

Remark: For M10 control (ZBD1) only values 0 and 1 are applicable.

 














Timeout Release


Time to enable the corresponding car call (button) after activation of the release signal (KKL) from car reader etc. When the time has elapsed the car call is disabled again. Used for access control functions ZBC3 and ZBD1 (see ESS)





















Card Check Mode M10


M10 card reader check mode. Used for access control function ZBD1. Defines when an authorized passenger must/can enter the required authorization (e.g. by inserting a badge into a card reader) in order to access the restricted destination floor/landing. This action will then activate the corresponding input (KKLC) on the elevator control.

1: Validation in the car required/possible when the car has reached the destination floor.
2: Validation in the car required/possible during the trip.
3: Validation the car required before the car starts to the destination floor.





















Card Reader PIN Config.


Used for parallel card reader interface (access control). The table specifies the accesses (floor and side) to be released after activation of a input KKLC-n.
Meaning of parameter fields:

 ¨01[06 04 ii kk zz nn]

ii: Index number for parameter table
kk:Floor number (1 to N)
zz: Entry side (0 = side all / 1 = side 1 / 2 = side 20)
nn: Pin number n of input KKLC-n

The default values of the table are set so that the KKLC input with pin number x release all accesses on floor x. (input KKLC has no side attribute)








Representation of default pin configuration in PCT:

   ¨7C                                  *   PCT_GP_CARDREADER
     {…
     ¨01 [06 04 01 01 01 01]            *    Entry 1, floor 1, side 1, P_CR_PIN_CFG PIN 1
     ¨01 [06 04 02 01 02 01]            *    Entry 2, floor 1, side 20, P_CR_PIN_CFG PIN 1
     ¨01 [06 04 03 02 01 02]            *    Entry 3, floor 2, side 1, P_CR_PIN_CFG PIN 2
     ¨01 [06 04 04 02 02 02]            *    Entry 4, floor 2, side 20, P_CR_PIN_CFG PIN 2
     ¨01 [06 04 05 03 01 03]            *    Entry 5, floor 3, side1, P_CR_PIN_CFG PIN 3
     …
     ¨01 [06 04 (2*N-1) (N) 01 (n)]     *    Entry (i), floor (N), side 1, P_CR_PIN_CFG PIN 2
     ¨01 [06 04 (2*N)     (N) 02 (n)]   *    Entry (i), floor (N), side 20, P_CR_PIN_CFG PIN 3
     }

i: Index for Entry
N: Number of Floors in this building
n: Max Pin Value defined with N_KKLC-n

Remark: In the default table, the card reader pins are allocated to the floors  with the same
number. Entries with floor numbers where no corresponding, individual card reader pin is
available, are completed with the highest card reader pin number configured in the PCT 
(see example above). In case of  diffrently specified access controled floors,  the 
allocation table has to be set up or edited corresponding to those specifications.
