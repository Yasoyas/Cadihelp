Firemans Control

Summary

 This section describes the parameters in the parameter window Firemans Control of the parameter group Services in subsystem Lift Control.

USA JBF Region

This Parameter defines the region or country with special, region dependent Fire service behaviour. It is used in core SW and in services (SCT)

Fire Code's are defined as following:
00: undefined
01: undefined (default, if the parameter is missing)
02: USA Chicago
03: USA Massachusetts
04: USA Region 1
05: USA A17.1-2000
06: USA New York City
07: USA A17.1-2007
08: USA A17.1-2000 (with variant P2RR)

Note:
This parameter is independent of pre A17-2000 and A17-2000. This behavior is handled in ANT_Code_Type.

Example of Parameter:
¨A2[ 02 02 0002 ] * P_STR_JBF_REGION = 2 (02, USA, CHICAGO)

Keep Door Open


Defines the door behavior after the fire emergency recall trip when the door(s) has openend on the Fire Recall Floor (also applies for the alternative Fire Recall Floors) according to the parameter 

0: The door(s) close(s) after it was open when the normal door open time has elapsed
1: The door(s) remains open

The door behavior is controlled with this parameter on the following fire recall services:

BR1/2/3/4 (standard)
BR1/3 BE
BR1/2/4 (ALT) HK
BR1/2 (ALT) SG
BR1/4 VN























JNFF/NS Priority Firefighter Lifts


Specifies which lifts have to be treated with priority in case of Emergency Power Service. Prioritized (fireman-) lifts are evacuated one after the other but before the non-prioritized lifts in RNO and immediately released for JNO after evacuation . With the appropriate bit set to 1 the lift is treated with priority while the bit set to 0 means standard setting (non-prio treatment respectively automatic selection). Normally one lift within a group is foreseen as fireman lift. The priority for fireman lifts overrides the normal JNO priority parameter P_STR_JNO_REL_LIFT.

Remark: Make sure the parameter P_STR_RNO_LIFTS_IN_OP is set to a value equal to the number of prioritized (fireman-) lifts plus one.
Remark: Same parameter value must be used on all lifts in order to prevent inconsistencies of parameter data (group parameter). The parameter is implemented as a bit mask with each bit representing a particular lift.
Remark: For TX4 the same function is done with the parameter P_STR_SERV_RESERVE_10

Bit 0 (lsb): Lift 1
Bit 1: Lift 2
...
Bit 9: Lift 10








Lifts 2 and 3 prioritized -> bit 1 and 2 set -> parameter value RNO_JNO_PRIO_LIFTS = 0x06

To be defined for NPE application













Light/Fan at open Door


Specifies if cabin light and cabin fan should be switched off or remain on after arriving on the emergency recall floor (for Emergency Power NSxx and BRx).

The parameter is only used on the following services:
NS1
NS11
BR1/2 (ALT) SG
BR1/2/4 (ALT) HK
BR1/4 VN
BR2 EN

For all other fire recall services the behaviour can not be altered with the parameter.
