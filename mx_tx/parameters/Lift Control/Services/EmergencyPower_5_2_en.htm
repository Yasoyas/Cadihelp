Emergency Power

Summary

 This section describes the parameters in the parameter window Emergency Power of the parameter group Services in subsystem Lift Control.

Evacuation Mode

Activation options are automatic or manually via a control center.

Activation Mode:
1 = AUTO: Evacuation automatically after the delay ZNO (selects the
best lift)
Set if GRP_NS_MODE = A

2 = A_M_OVER: Evacuation automatically with manual override.
(USA-version)
Not supported by COS

3 = GLT_GROUP: Group evacuation is controlled by the building
interface GLT by the group input RFEFG for evacuation and RFNOG
for release of elevators (selects the best lift)
Set if (GRP_NS_MODE = M) and ((GRP_BUILDING_SUPERV_INTERFACE = GROUP)
or (GRP_BUILDING_SUPERV_INTERFACE = LAG))

4 = GLT_LIFT: The evacuation is fully controlled by a building-
interface GLT. The evacuation start per lift is
given by the lift input RFEF.
Set if (GRP_NS_MODE = M) and (GRP_BUILDING_SUPERV_INTERFACE = LIFT)


>> Rule for MVX-K:
Since signals RFEFG and RFEF are never configured, evacuation is always
automatic (COS characteristic GRP_NS_MODE is not relevant):

Set always to 1.

Evacuation Delay


Used for emergency power evaucation RNO.

RNO_ZNO (Zeit Notstrom): Time delay to start the (group) evacuation after activation of input RNO. The value for ZNO is limited to a minimal value (when ZNO > 0) for system recovery after power is restored with RNO.

With value 0 the evacuation delay is infinite. The evacuation must then be triggered with the input RFEF or RFEFG. The evacuation delay time (ZNO) is also used with building evacuation in order to start the emergency power evacuation within a group if the evacuation signal is not activated.


Values for ZNO in case of building evacuation:

       RNO_ZFNO
       +----------------------------------------------------+
       |     group 1                       group 2          |
     +-v---------------------+       +------------------+   |
     | +----+----+----+----+ |       | +----+----+----+ |   |
     | | L1 | L2 | L3 | L4 | |------>| |L1 | L2 | L3  | |---+
     | +----+----+----+----+ |       | +----+----+----+ |
     +-----------------------+       +------------------+
     ^                               ^
     |                               |
     RNO_ZNO(1)                      RNO_ZNO(2)

     Group 1: RNO_ZNO =10s (fix)
     Group 2: RNO_ZNO =(nbr_lift_G1 + 1)*T1_G1 + T2_G1 + RNO_ZNO_G1
     Group 3: RNO_ZNO =(nbr_lift_G2 + 1)*T1_G2 + T2_G2 + RNO_ZNO_G2 + RNO_ZNO_G1

Abbreviations:
--------------
T1_G1 = Evacuation delay for one lift in group 1. It correspond to parameter "RNO_TIMEOUT_EVAC_DONE"

T2_G1 = The given time to evacuate the group 1 in a second step. It correspond to parameter "RNO_TIMEOUT_GROUP_EVAC"

>> Rule for MVX-K:
  Always set to FieldValue (default).





















Timeout Evacuation Lift


For Emergency power service, specify time to wait before blocking a lift that has not done evacuation as required.

>> Rule for MVX-K:
  Always set to FieldValue (default).





















Timeout Evacuation Group


For emergency power service RNO. Defines the time to wait until the group evacuation is considered completed when (after a round of evacuation attempts) one or more lifts have not completed the evacuation to the emergency power recall floor or the closest floor.

>> Rule for MVX-K:
  Always set to FieldValue (default).





















Evacuation Floor


Parking floor by service NS11 / NS21 (Emergency power evacuation / Notstromevakuation)

Important: when Emergency power and Fire services are present, the parking floor for both services must be the same.

>> Rule for MVX-K:
  if LOC_NS_EVAC_STOP = not ordered then set to LOC_MAIN_FLOOR
  else set to LOC_NS_EVAC_STOP
  (cSTOP_NS = LOC_NS_EVAC_STOP, cSTOP_MAIN_FLOOR = LOC_MAIN_FLOOR)





















Open Doors Evacuation


Defines which door (front side or/and rear side) must open at the emergency power recall floor. Used for emergency power services with evacuation to predefined recall floor (NS11 / NS21).

Door side mask:

1 = ZS1
2 = ZS20
3 = ZS1 + ZS20

>> Rule for MVX-K:
  if (LOC_NS_EVAC_STOP = not ordered) then set to FieldValue (default)
  else set according to ACCESS value of LOC_NS_EVAC_STOP, using above
  door side mask scheme





















JNO Release Mode


Defines how the elevator is released for emergency power operation (JNO).

  Activation Modes:

      1 = MANUAL: Manual release for normal operation under emergency power with the key-switch JNO per car. The number of lifts which can be released simultaneosuly is limited by the parameter "Lifts in Operation". If the parameter RNO_ACTIVATION_MODE is set to GLT_GROUP the input RFNOG must be active before a lift can be switched to JNO mode.
      Set if GRP_NS_MODE = MANUAL 

      2 = AUTOMATIC: Automatic release for normal operation under emergency power (best lift). The number of lifts released simultaneously is limited by the parameter "Lifts In Operation". If the parameter RNO_ACTIVATION_MODE is set to GLT_GROUP the input RFNOG must be active before a lift can be switched to JNO mode.
      Set if GRP_NS_MODE = AUTO and GRP_BUILDING_SUPERV_INTERFACE =  GROUP

      3 = A_M_OVER: Release lift(s) automatically with manual override.
          The number of lifts released depends on the parameter "Lifts
          In Operation".
      Not supported by COS 

      4 = GLT_LIFT: Release lift(s) only from the building interface
                    GLT with the input lift RFNO.
          The number of lifts released is fully controlled from the
          building interface.
      Set if GRP_NS_MODE = AUTO and GRP_BUILDING_SUPERV_INTERFACE = LIFT

>> Rule for MVX-K:
  Since signal RFNO is not present, only values 1 and 2 apply
  (COS characteristic GRP_BUILDING_SUPERV_INTERFACE is not relevant):

  Set to 1 if GRP_NS_MODE = MANUAL or not ordered (default)
  Set to 2 if GRP_NS_MODE = AUTO

Also known as:
P_STR_JNO_RELEASE





















JNO Delay


RNO_ZFNO: RNO Zeit Freigabe Notstrom

It is a delay to release a group back into service JNO after finishing the evacuation of all groups. It is only used in case of  building evacuation. See also parameter RNO_ZNO and JNO_RELEASE

Value: 0 means infinite, used in case of manually release only with input group RFNO.

RNO_ZFNO = RNO_ZNO last group + (nbr_lift_Gn + 1 )*T1_Gn + T2_Gn

T1_Gn: Evacuation delay for one lift in the last group. It correspond to parameter "RNO_TIMEOUT_EVAC_DONE"

T2_Gn = The given time to evacuate the last group in a second step. It correspond to parameter "RNO_TIMEOUT_GROUP_EVAC"

>> Rule for MVX-K:
  Always set to FieldValue (default).





















JNO Release max.No.Lifts


Defines how many lifts can operate simultaneously with JNO service after evacuation sequence has completed during Emergency power service NS2/21.
Remark: The same parameter value must be set on all lifts in order to prevent inconsistencies of order data (group parameter). The parameter has no impact on the number of lifts evacuated simultaneously ( always one at a time - refer also to P_STR_RNO_JNO_PRIO_LIFTS ) 

For NA:  max. value = 3 when there is no interlocking

>> Rule for MVX-K: 
  if (LOC_NS_EVAC_LIFT = not ordered) then set to FieldValue (default)
  elsif (LOC_NS_EVAC_LIFT = ALL) then set to GRP_NUMBER_OF_LIFTS
  else set to number of lifts ordered in LOC_NS_EVAC_LIFT.

For NA: max 3 when there is no interlocking





















JNO Release Lift Priority


Defines which lifts are released for service JNO when the parameter 'JNO Release Mode' (PCT name: JNO_ACTIVATION_MODE) is set to automatic release. The parameter has no impact when the 'JNO Release Mode' is set to 'Manual'.

The parameter consists of eight values representing priority values 1 to 8. Each lift can be assigned to a JNO release prority. Table must be filled starting from priority 1 (first entry in table). 
Zero values must be at the end. If a value 0 is in the field 2 and all following fields contain 0 the best lift to be released for JNO will be choosen automatically.
Lifts released for JNO is limited by parameter P_STR_RNO_LIFTS_IN_OP


>> Rule for MVX-K:
  Always set to "1 0 0 0 0 0 0 0 0" (default).





















JNO Release Lift Priority Feeder 2


It specifies the order of put back priorities of the feeder 2 lifts.





















Lifts on Feeder 1


It specifies what lifts are on emergency power feeder 1. The lift id of the first car must be at index 2, the next id at index 3 an so forth. The lift ids after the last used index must be 0 if there are lifts on feeder 2. If there are not 8 cars on feeder 1 and there are no lifts on feeder 2 then it's better to have the lift ids of the not used indexes set to 0.








example:

¨85[ 06 09 01 01 02 00 00 00 00 00 00 ]    * P_STR_RNO_LIFTS_ON_FEEDER_1 = 1 1 2 0 0 0 0 0 0
Lift 1 and 2 is on Feeder 1














Lifts on Feeder 2


It specifies what lifts are on emergency power feeder 2. The lift id of the first car must be at index 2, the next id at index 3 an so forth. The lift ids after the last used index must be 0.

example:
¨86[ 06 09 01 03 04 00 00 00 00 00 00 ]    * P_STR_RNO_LIFTS_ON_FEEDER_2 = 1 3 4 0 0 0 0 0 0
Lift 3 and 4 are on Feeder 2





















Interlocking


Type of interlocking for multibank emergency power.

0: no multibank interlocking.
1: multibank interlocking with one car remaining in service.
2: multibank interlocking with up to three cars remaining in service.

Remark: commissioning, edit and reset not defined

example:
¨45[ 02 02 0000 ]    * P_STR_RNO_INTERLOCKING = 0





















Interlocking Feeder 2


Type of interlocking for multibank feeder 2 emergency power.

0: no multibank interlocking.
1: multibank interlocking with one car remaining in service.
2: multibank interlocking with up to three cars remaining in service.

Remark: commissioning, edit and reset not defined
example:
¨46[ 02 02 0000 ]    * P_STR_RNO_INTERLOCKING_2 = 0

The parameter is only used on the following services:

NS11

Emergency Power Manual Mode


Parameter to define the use of the Auto/Manual switch for EMP operation by the software.

In Emergency Power Operation (RNO) the control will read the status of the Auto/Manual switch for EMP operation only if this parameter is set to FALSE (0). If the parameter is set to true (1) the control will ignore the Auto/Manual switch for EMP operation and will assume that the switch is ON (manual).

If the parameter not in the PCT the control assumes the parameter is set to FALSE.
