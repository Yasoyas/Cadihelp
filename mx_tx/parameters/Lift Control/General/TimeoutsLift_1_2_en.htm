Timeouts Lift

Summary

 This section describes the parameters in the parameter window Timeouts Lift of the parameter group General in subsystem Lift Control.

CC Registr.Reserv Time

For KA/KS:
Specify the time to wait after a floor call has been served and the doors are closing before clearing the preselected direction to allow people in the cabin to give a car call.

For PI:
Specify the time to wait after door closed for the cabin to be free.

- For PI with load measuring: this time must be set to 1 ( 1 * 100ms = 0.1s)

- For PI without minimal load detection (load measuring): the time must not be less than 2 seconds (20 * 100ms = 2s). This is necessary because without minimal load detection it is not possible to know if somebody is in the car or not. If this time is too short a floor call may be allocated to the lift before the person in the cabin has given a car call.

 














Delay Car Fan Off


Defines the behavior of the car fan (RVEC-A). The fan is controlled by the switch JVEC on the car operating panel and the timeout value defined with the parameter.

If the input JVEC for fan switch on the COP is not configured the control of the fan is defined by the timeout parameter. If the value is 0, then the fan will be switched on when needed (e.g. during a trip) and immediately switched off afterwards. If the timeout has a different value (default 12000 i.e. 20 minutes), then the fan will remain on for the specified time.

If JVEC is present, the fan operation will depend on the state of the switch and the timeout parameter. 

When JVEC is off, the fan can not be switched on at all. When JVEC is switched on, a test is made for the timeout parameter and if equal to 0, then JVEC has full control of the fan and the fan will be switched on for the time that JVEC remains on. If the timeout parameter has a different value, then the fan will be switched on for the time specified in the parameter. Once the time has expired, the fan will be switched off, but it may be reactivated by either the service or the lift manager if so needed. As long as JVEC is on, this secondary control is possible. If JVEC is switched off at any time, the fan is switched off immediately and not allowed to be switched on again.

The service manager can specify that the ventilator should be switched on permanently or automatically or decoupled from the automatic switching and switched off. This will happen if JVEC is either not present or when it is  switched on and the timeout parameter is set to a value other than 0.





















Delay Car Light Off


Time (s) to wait before switching off the cabin light (output: RLC) when the elevator has no destination/direction and the doors are closed. When the parameter is set to zero the car light (RLC) switches off without delay.

This parameter was renamed (old name: P_STR_DELAY_CABIN_LIGHT)





















Master Check


Time interval between two cycles of the master check (check if there is one and only one master). Do not change the parameter in the field unless instructed to do so; always use the value defined in the PCT.

Remark: this value should always be higher than two times the value of the parameter P_STR_TIMEOUT_SEND_LIFT_ALIVE.

For MX-GC controls the value for this parameter was increased from 3.5s to 15s in order to avoid unnecessary master switching after resets (Oct 2002).





















New PI Floor Call Group


Time to wait before creating a new group of floor calls. Used for non-directional floor calls only (PI).  This is used for a special feature of PI that optimize the PI algorithm. A value of zero for this parameter means that this optimization is disabled.





















Check Valid Floor Calls


Defines the time interval for the periodical check if the registered floor calls are still valid. Valid means that a floor call can be served by an elevator in the group. Floor calls which can no longer be served are cancelled. The check is done according to the 'served floor table' which defines the served floors of each elevator in the group.





















Observe Traffic


Time intervall between two cycles of type of traffic determination (interfloor, up, down or noon peak).





















Board Rate Calculation


Interval between two boarding rates calculation cycles.





















Board Rate Decrement


Time interval after which the boarding rates must be decremented of one person.





















Trip Start


Timeout used for the monitoring of the drive start. If after a 'start_ok' command to the drive (which means that the door is locked) the drive does not start to move within this time the control will send the 'start_ok' again.





















Send Lift Alive


Time period after which the control sends the 'lift alive' message to the master. A value of 10 means that the elevator control repeats the 'alive' message every second. Do not change the parameter in the field unless instructed to do so; always use the value defined in the PCT.

Remark: this value should always be less than half of the value of the parameter P_STR_TIMEOUT_MASTER_CHECK.

For MX-GC controls the value for this parameter was increased from 1.0s to 4.0s in order to avoid unnecessary master switching after resets (Oct 2002).





















Timeout Recover Powerup


Maximum time allowed to recover door and drive when normal mains power is restored (power up). The control software expects the system to be ready within this time (does not include the time required for the software restart). Drive and door(s) remain blocked until drive and the door(s) are available for operation and the software can proceed to determine and activate the appropriate function (the behavior of door and drive is then controlled by that function/service e.g. 'N' etc.). 
In case the timer elapses the control allows door movements in order to release passengers provided the car is leveled on a floor.

This is oneshot timer used only after power cycle.
