Lift General

Summary

 This section describes the parameters in the parameter window Lift General of the parameter group General in subsystem Lift Control.

Lift ID

Identification number of this lift. The first lift of the group has to be the number 1.

Attention: The lift number must be unique on all elevators operating within a group.

 














Lowest Floor-Lift


Defines the group related floor number of the lowest floor served by the lift. 
(The parameter is also used by the drive to allow it to send the right selector number for the group.)

Caution: make sure the value is set according to the definition made in the table 'served floors' for the lift (see group parameters).









Example: triplex with 3 floors

        L1  L2   L3    
       +---+---+       
    4  |   |   |       
       +---+---+---+   
    3  |   |   |   |   
       +---+---+---+   
    2      |   |   |   
           +---+---+   
    1      |   |   |   
           +---+---+   

 => lowest floor  L1 = 3
                  L2 = 1
                  L3 = 1













Highest Floor-Lift


Defines the group related floor number of the highest floor served by the lift. 

Caution: make sure the value is set according to the definition made in the table 'served floors' for the lift (see group parameters).








Example: triplex with 4 floors
                                       
    L1  L2   L3    
       +---+       
4      |   |       
       +---+---+   
3      |   |   |   
   +---+---+---+   => last floor  L1 = 2
2  |   |   |   |                  L2 = 4
   +---+---+---+                  L3 = 3
1  |   |   |   |   
   +---+---+---+













Number of Side


Number of doors (entrance sides) for this lift.





















Commission Number


Commission number of this lift (7 digits). Each lift of the group has its own number.





















Max.Nbr.Passengers


Maximum number of persons which can occupy the car. The default value (nominal load divided by load per person) is normally ok. With M10 control this parameter is used for the allocation of cars to M10 destination calls. If the default value is not ok (not enough room for the number of person), M10 will always allocate too much people for this car. For M10 the value must not exceed 32.

Default value: P_LFT_NOMINAL_LOAD / P_STR_LOAD_PER_PERSON





















Country Features Enable


To enable special country features.

Country features id:
0: no special features (standard software)
1: USA features enabled





















LON Bus Architecture


Defines type of user bus architecture (IO's). 

Default value (Value = 0 or parameter 'USERBUS_ARCHITECTURE' not defined in PCT):
LON group bus connection between all elevator controls within one group.
Each elevator control has two separate buses (channels): one for lift I/Os and one for group I/Os. This architecture is used with TX-GC / 10-GC / MX-GC (up to R3).

Value = 1: 
LON group bus connection between all elevator controls within one group.
Each elevator control has two separate buses (channels): one for lift I/Os and one for group I/Os. This architecture is used with TX-GC / 10-GC / MX-GC (up to R3).

Value = 2:
No LON group bus connection between the elevator controls within one group.
Each elevator control has only one bus down the shaft for lift I/Os and for group I/Os.

Rem: The parameter is used to define the bus architecture on MX controls. On TX-GC/GC2 controls the user bus architecture is defined by the system type.





















Manual Car Call Cancel


Mode for manual cancelling of car calls

0: OFF
1: Cancel car call when car call button is pressed twice

refer also to parameter P_STR_MAN_CC_CANCEL_TIME






















Maximum Time Cancel Car Call


Maximum time allowed between two activations of the call button to trigger car call cancelling. Only used when manual cancelling of car calls activated (refer also to parameter MAN_CC_CANCEL)






















Block Car Call Next Floor


Disable registration of car call for next floor - for floor below and above position of car.

0: OFF
1: Do not register car call for next floor below and above car position























Car Fan Mode


Defines the behavior of the car fan (output signal RMVEC-A) based on status of the car fan switch JVEC. The following modes can be set:

0: No car fan
1: Automatic: No car fan switch JVEC available. The fan is automatically activated during trips with adjustable off delay.
2: OFF/ON: Switch JVEC used to turn the car fan on/off.
3: OFF/Automatic: With JVEC = OFF fan is off. With switch JVEC = ON the car fan is automatically activated during trips with adjustable off delay.
4: ON/Automatic: With switch JVEC = ON the fan is switched on permanently. With JVEC = OFF the fan is automatically activated during trips with adjustable off delay.
5: OFF/ON/FAST: For 2 speed car fan only (output RVECF)
6: Toggle ON/OFF: Activation of input JVEC switches fan ON when it is OFF or switches fan OFF when it is ON.





















LON Freeze Node Command


Defines if the control requires manual LON freeze during commissioning with SMLCD/SPECI/CADI)

0: No LON freeze command available on service user interface.
1: LON freeze command available on service user interface.

Remark: the parameter is only valid/required for TXR5 and GX with common release 2.10





















Number Of Decks


Number of decks of elevator car

1: Single Deck Elevator (Default)
2: Double Deck Elevator
