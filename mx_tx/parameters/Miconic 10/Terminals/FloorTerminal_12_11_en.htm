Floor Terminal

Summary

 This section describes the parameters in the parameter window Floor Terminal of the parameter group Terminals in subsystem Miconic 10.

Blinking Alloc. OFF Duration

OFF phase duration (time) when allocation is displayed (blinking) on terminal.

For terminals with software versions:
'Z Line 1' version >0601
'Z Line 2' version >0701
the beeper is controlled together with the (blinking) allocation display. The beeper is OFF according to the allocation display on the terminal (parameter BLINK_ALLOC_OFF_DUR).

 














Blinking Alloc. ON Duration


ON phase duration (time) when allocation is displayed (blinking) on terminal.

For terminal with software versions:
'Z Line 1' version >0601
'Z Line 2' version >0701
the beeper is controlled together with the (blinking) allocation display. The beeper is ON according to the allocation display on the terminal (parameter BLINK_ALLOC_ON_DUR).





















Next Digit Timeout


Timeout to wait second ‘floor name’ digit on terminal when giving an M10 call.





















Nbr.of Digits per Code


Required number of digits for the service codes entered from a M10 terminal.

2.1.2005 L.Barmet: minimum value changed from 2 to 3





















Allocation Display Duration


Duration of allocation display on terminal.





















Call Display Duration


Duration of the M10 call display on terminal.





















Lift Names


Designation of cars within group for M10. Defines the character displayed on the terminal to indicate which lift the call was allocated to.





















Distances Terminal/Lift


Defines the distance in cm between each car (1-8) and each riser (terminal position). It is used to calculate the walking time from the terminal to the car. The walking time is taken into consideration when assigning a Miconic 10 call. 

CAUTION: this parameter table should always be complete in the PCT. It means all indexes should be present (0 to 23, 24 ‘lines’).
PCT generation for MVX-K: please refer to instruction document J275101.

Since GC Software Version 1.41: 
If you set "distance to car" for a specific lift to the maximum value of 4000cm, this lift will never be assigned.

Position Terminal/Lift


Defines the direction/location of each car (1 to 8) related to each riser (terminal position). Used to select the direction sign displayed with the car name on the terminal when an allocation is done.

Direction to the car:
  1: in front left       
  2: in front            
  3: in front right     
  4: behind left
  5: behind
  6: behind right

Special Service Codes


For the de-/activation of commission specific services from the destination terminal with a dedicated code (in addition to the predefined standard service activation codes which are NOT part of the PCT).

The parameter allows to define 32 special codes to be used for commission services. For each code the parameter defines:

1. The service-ID of the Service to be activated
2. The Lift-ID on which the Service has to be activated
3. The information if an additional M10-call on the terminal is required 
4. If the code activates or deactivates the service

Any of these M10-codes entered on the terminal will set the service-variable (SERV_VAR_37_CODED_SERVICE) which can be used to activate the commission-service.









Example:
¨69   **  PCT_GP_M10_DEVICE
  {
  ...
  ¨0D[06 05 012E010001]   ***  P_M10D_CODED_SERVICE = 1 46 1 0 1
