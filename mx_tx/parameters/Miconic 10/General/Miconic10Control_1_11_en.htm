Miconic 10 Control

Summary

 This section describes the parameters in the parameter window Miconic 10 Control of the parameter group General in subsystem Miconic 10.

Max. Lifts on Special Service

In order to avoid low traffic handling performance of the elevator group because too many elevator running with active special services the maximum number of elevators running with active special service must be controlled. The parameter defines the maximum number of lifts in the group with active special service simultaneously. Limitation is only valid for services JRV / JRVP / JAB / DDF1 / NF2 / JNO / NF1 / JLI triggered from a destination terminal.

 














Destination Call Limit


Limit for allocation of coincident destination calls entered on a floor.

This feature reduces the negative impact for the elevator group (traffic load) if impatient or undisciplined passengers enter the same call several times on the same terminal and this way enter/book unnecessary trips. If the number of coincident calls entered consecutively at a particular floor exceed the call limit value, the calls are no longer considered in the trip-planning/cost-calculation. The call acknowledge and the signalisation are done normal. The similar call limit can be set per floor. 
For the Main Floor or other busy floors the call limit should be set higher than for normal floors. The Call Limit function is disabled by setting the limit value (per floor) to 32.

This feature has to be used with caution because it violates the M10 rule that each destination call entered represents one passenger and a booking for a trip. There is no way to distinguish unnecessary (nuisance) calls given by one person from legitimate calls given by different people arriving at the same floor at the same time who want to travel to the same destination. When the feature is enabled it can happen that a group of passengers who operate the lift interface correctly get a car with not enough space for all of them or that other people that booked a trip from another floor will see a full car stopping at their floor.

M10 Subgroup


M10 subgroup. It defines for each Mode which car accepts calls from which Riser.
It is possible to define 8 differents Modes. Each parameter line corresponds to one Mode. Each parameter contains a 8bits mask per Riser representing the cars that can serve a call from this Riser.








Car mask:
bit#         | 7 | 6 | 5 | 4 | 3 | 2 | 1 | 0 |
-------------+---+---+---+---+---+---+---+---+
car#         | 8 | 7 | 6 | 5 | 4 | 3 | 2 | 1 |
-------------+---+---+---+---+---+---+---+---+
value to add |128| 64| 32| 16| 8 | 4 | 2 | 1 |
