Door General Side 1

Summary

 This section describes the parameters in the parameter window Door General Side 1 of the parameter group Side 1 in subsystem Doors.

Supress VST-S

When the parameter is set the door control software will switch off (suppress) the output VST-S as soon as the door reaches the door end contact KET-S (input). This parameter is mainly used for 3rd party doors (e.g. Wittur) to avoid problems if door drive VST-S remains ON after door is fully closed.

 














Door Type


Specify door type per floor installed on respective side. 
This parameter is a table with floor number as index.
------------------------------------------------------------------------
TX-GC:
Byte 1: Floor number (index)
Byte 2: Car/Shaft Door Type
  Automatic doors:
  1 = QKS9, V30-230, V30-400( V50-230, V50-400, FERMATOR, QKSW DC, QKSW 400AC, QKS9VF, QKSW_RCF1)
  2 = QKS10  
  3 = QKS11 
  10 = Heavy auto door with additional servo motor
  Semi-Automatic Doors:
  30 = QK11
  Manual Door:
  50 = Hand Door (RTS)
Byte 3: Floor Access Control
   0 = no shaft door
   1 = normal access to floor with corresponding car call or DT-O
   2 = door exists / access to floor only with key switch (JDC) for the corresponding car call /  DT-O disabled when door closed
--------------------------------------------------------------------------------------------------------------------
MX-GC / TX-GC2:
Byte 1: Floor number (index)
Byte 2: Shaft Door Type
  100 = No Shaft Door 
  101 = Manual Shaft Door
  102 = Automatic Shaft Door (moved by car door)
Byte 3: Floor Access Control
   0 = no shaft door (only when Byte 2 = 100)
   1 = normal access to floor with corresponding car call or DT-O
   2 = access to floor only with key switch (JDC) for the corresponding car call /  DT-O disabled when door closed
--------------------------------------------------------------------------------------------------------------------
