Door Times Side 1

Summary

 This section describes the parameters in the parameter window Door Times Side 1 of the parameter group Side 1 in subsystem Doors.

Min. Door Open Time

Defines the minimum time the door remains open after reversing or normal opening. The door close button (DT-S) is disabled during this time.

 














Min. Door Open Time Hall Call (ADA)


Defines the minimum time the door remains open when serving a landing (hall) call on handicapped elevators. The door close button (DT-S) is disabled during this time. Used for elevators which have to fulfill the ADA requirements for handicapped people. See ADA accessibility guidelines (ADAAG) chapter 4.10.7

4.10.7* Door and Signal Timing for Hall Calls. 
The minimum acceptable time from notification that a car is answering a call until the doors of that car start to close shall be calculated from the following equation:
   T = D/(1.5 ft/s)    or    T = D/(445 mm/s) 

where T total time in seconds and D distance (in feet or millimeters) from a point in the lobby or corridor 60 in (1525 mm) directly in front of the farthest call button controlling that car to the centerline of its hoistway door (see Fig. 21). For cars with in-car lanterns, T begins when the lantern is visible from the vicinity of hall call buttons and an audible signal is sounded. The minimum acceptable notification time shall be 5 seconds. 























Hold Open Exiting


Normal door open time when a car call is served. For side 1 only. The door open time can be reduced by pressing the door close button.

old parameter name: P_STR_DC_DELAY





















Extra open time Floor_Car calls


Additional door open time for entrance side 1 when a car call (DC) and a floor call (DE) are served concurrently. The parameter value is added to the normal door open time defined by the parameter for the door open time for side 1 when a landing call is served ('hold open boarding').





















Reopen Delay


Delay time after door drove from one direction until it drives it the other. The delay time is normally 100ms (0.1s). A higher delay time reduces noise and mechanical forces (is important for sensitive door types to increase their life), but increases also the door times (flight time). The influence of this parameter can be observed when the door reverses (reopening).





















Final Timer


Specifies the time to wait after a door close command before activating the final timer (door nudging) if the door does not close. The final timer function disables the reversing devices, activates the buzzer in the car and closes the door with reduced speed (the closing force limiter (KSKB) remains active). 
The final timer function is deactivated by setting the parameter to 0.





















Hold Open Boarding


Normal door open time (side 1) when a landing call is served (boarding). The door open time can be set individually for each landing. (The actual time the door remains open be reduced by pressing the door close button.)








Example: 
+------+-----------+
| FLOOR| OPEN_TIME |
+======+===========+
|  1   |    40     |
+------+-----------+
|  2   |   100     |
+------+-----------+
|  3   |    40     |
+------+-----------+
|  4   |    40     |
+------+-----------+













Door lock timeout


Automatic doors: Defines the door unlock timeout. If the door is locked at standstill the closing signal (VST-S) remains activated until the timeout has occured or a block door command has been received.

Manual operated doors: Defines the door unlock timeout. If the door is locked (VRV) at standstill the locking device remains energized until the timeout has occured or a block door command has been received.

Values:
0:	delay time for unlock timeout is 30 sec. (Default value)
1 - 36000:	delay time for unlock timeout 0.1 ... 3600 Sek.
> 36001:   no unlock timeout (door remains locked)





















Time VST-O off


Delay time to switch the output VST-O off after the door has reached the open position (KET-O active)

0: no delay time for VST-O off (default value)
1 to 31999	: delay time for VST-O off in 100ms (0.1 ... 3199 sec.)
32000: VST-O remains active when door is open. (until door close command)
65535: Load default value (-> RC software will set parameter to 0)

Zeit Verzögerung für Abschaltung von VST-O
Bei einigen Türantrieben wird verlagt, dass VST-O am Türantrieb anliegt, solange die Tür ein Öffnungskommando hat. Typisch für eine solche Anwendung ist z.B. die 'Tormax' Tür, welche automatisch über eine Federkraft schliesst.

(Die Steuerung erfolgt über folgendend RC Parameter:
P_KAB_RESERVE_5    =    P_KAB_KEEP_VSTO_TIME)





















Max Door Travel Time


This parameter defines the maximum time the software waits for the door end contacts to be reached. This means if the door is moving from close to open, then the door open contact should be active within maximum this time. This parameter need not be set essentially in the PCT, nor does it need to be present. In case of a 0 or this parameter not existing in the PCT, the core software takes a default of 15secs. Any non-zero value present in the PCT, would be taken as is for this monitoring.





















Back Step Time


This parameter defines the time for which the door should continue reversing in case KSKB is activated when the lift is outside the door zone and the door is closing. 
1 unit equals 100ms
