Peak Control General

Summary

 This section describes the parameters in the parameter window Peak Control General of the parameter group Peak Control in subsystem Group Control.

Load Car Call Delay

Delay before allowing the load car to leave the lobby when a car call is registered or when a floor call is allocated. Used for up-peak and next car. See also P_STR_LOAD_CAR_LOAD_LIMIT.

 














Load Car Call Limit


Load limit to reach to allow the load car to leave the lobby. Used for up-peak and next car. See also P_STR_LOAD_CAR_CALL_DELAY.





















Next Car On


Next car feature available or not.
 Mode:
  0 = OFF
  1 = ON





















Start High Zone


First floor of the high zone in zoned up-peak.

Remark: zoned up-peak not yet implemented.





















Load Limit Quota Modify


Load limit of all car in quota leaving the lobby to modify number of car in quota for up-peak. When all the cars currently part of the quota leave the main floor with a load over the one specified by this parameter and the maximum quota has not been reached, the quota will be incremented by 1. If all the cars currently part of the quota leave the main floor with a load less than the one specified by this parameter minus 20 % (P_STR_UPK_LOAD_LIMIT_QUOTA_MODIF - 20 %) and the minimum quota (2) has not been reached, the quota will be decremented by 1.





















Up-Peak Mode


Enable/disable up-peak operation (for all activation means).

Up peak traffic handling mode:
0 = always off, up-peak not allowed (disabled)
1 = normal, up-peak is allowed (enabled)





















Activation Nbr.Car Lobby


For up-peak activation condition: number of cars leaving the lobby. See also P_STR_UPK_ACT_LOAD and P_STR_UPK_ACT_NB_CC.
Up-peak is activated when a number of cars (P_STR_UPK_ACT_NB_CAR_LOBBY) leave the lobby (main floor) fully loaded (P_STR_UPK_ACT_LOAD) within a certain time interval (fixed, 2 minutes) with a minimum number of car calls (P_STR_UPK_ACT_NB_CC).





















Activation Load Limit


For up-peak activation condition: load limit of the cars leaving the lobby. See also P_STR_UPK_ACT_NB_CAR_LOBBY and P_STR_UPK_ACT_NB_CC.





















Activation Nbr.Car Call


Used for the automatic activation of up-peak traffic handling. 
Activation condition: minimum number of car calls registered when the car leaves the lobby. 
See also  P_STR_UPK_ACT_NB_CAR_LOBBY and P_STR_UPK_ACT_LOAD.





















Activation Nbr.Full Cars


For down-peak activation: number of cars reaching full load in the down direction limit. See also P_STR_DPK_ACT_RATIO and P_STR_DPK_FULL_LOAD_BYPASS.

Down peak is activated when the number of cars reaching full load (P_STR_DPK_FULL_LOAD_BYPASS) in the down direction exceeds the limit specified by P_STR_DPK_ACT_NB_FULL_CARS within a fixed time interval of 2 minutes, and the ratio of down-boarding rate to the up-boarding rate exceeds the limit specified by P_STR_DPK_ACT_RATIO.

Old (before June 2002): There is no parameter to choose the availablity of down-peak feature. This is performed indirectly using the activation parameter P_STR_DPK_ACT_NB_FULL_CARS. If this parameter is set higher than 100 the down-peak feature will never be activated and therefore is not available.

New: there is now a down-peak mode parameter (P_STR_DPK_MODE) to enable/disable down-peak operation for all activation means and another parameter (P_STR_PEAK_ACTIVATION_MODE) to enable/disable each activation means for all peak programs.





















Activation Ratio


For down-peak activation: limit of ratio of down-boarding rate to up-boarding rate. See also P_STR_DPK_ACT_NB_FULL_CARS and P_STR_DPK_FULL_LOAD_BYPASS.





















Full Load Bypass


Full load limit to bypass floor calls when in down-peak. This limit is generally set higher than the normal full load (P_STR_LOAD_VALUE_FULL_LOAD) in order to obtain better filling of the cars.





















Number of Load cars


Number of cars which are allowed for simulultaneous boarding at the lobby during up-peak mode after returning to the lobby floor. 

The maximum value is the number of  cars in the group minus one.

Remark: This value should not be set too high because people will start to board on different cars and cars will eventually not be filled as required during up-peak. This will again result in longer waiting times for other passengers.








Example:  2 cars simultaneously

      ¨5C                       *   PCT_GP_RS_PEAK
       {
         ...
       ¨A3 [02 02 0002]         *    P_STR_UPK_NB_LOAD_CAR
         ...
       }














Number of UP-Peak car at Start


Number of car taken part of up-peak when up-peak starts. 
The default is half of the available car (but not less than 2).
The minimum is 2 because up-peak with only one car makes no sense.
The maximum is 7 (maximum nb cars in group - 1) because at least one car should stay out of the quota to serve the down calls.









Example:  4 cars in quota at start (default for a group of 8 cars)

      ¨5C                       *   PCT_GP_RS_PEAK
       {
         ...
       ¨A4 [02 02 0004]         *    P_STR_UPK_START_NB_QUOTA_CAR
         ...
       }














Load Car Side


Entrance side for the boarding of the car (both sides not allowed). Used for automatic opening of the door during up-paek at the main floor.









Example:  Load car opens door side 1
      ¨5C                       *   PCT_GP_RS_PEAK
       {
         ...
       ¨A5 [02 02 0001]         *    P_STR_LOAD_CAR_SIDE
         ...
       }














Load Car Direction


Direction for the Load car.

In up-peak mode it is the direction the Load car will signal (LW) at Main floor.
If direction down is chosen, the up-peak will be done in the down direction (required with the lobby at top floor).









Example:  Load car direction up
      ¨5C                       *   PCT_GP_RS_PEAK
       {
         ...
       ¨A6 [02 02 0001]         *    P_STR_LOAD_CAR_DIR
         ...
       }














Down-Peak mode


Enable/disable down-peak operation (for all activation means).

Down-peak traffic handling mode (Software reference: P_DPK_MODE):
0 = always off , down-peak not allowed (disabled)
1 = normal, down-peak is allowed (enabled)









¨5C            *   PCT_GP_RS_PEAK
 {...
  ¨A7 [02 02 0000]   *    P_STR_DPK_MODE
  ...
 }














Peak Activation mode


Used to define/enable the activation mode (trigger) for the peak programs. The possible activation modes are:  Manual, Time controlled and Automatic.

Values: 
0 (000bin) = PK_NO_ACTIVATIONS
1 (001bin) = PK_AUTO
2 (010bin) = PK_TIMER
3 (011bin) = PK_TIMER_AUTO
4 (100bin) = PK_MANUAL 
5 (101bin) = PK_MANUAL_AUTO
6 (110bin) = PK_MANUAL_TIMER
7 (111bin) = PK_ALL_ACTIVATIONS

Defined with a bit mask:
Bit 0 (LSB) enables the automatic activation
Bit 1 enables the timer activation 
Bit 2 enables the manual activation.

Manual activation has priority over timer activation.
Timer activation has priority over automatic activation.
The automatic activation is based on the observed traffic situation.
