Templates

Summary

 This section describes the parameters in the parameter window Templates of the parameter group Peak Control in subsystem Group Control.

Day Template 1

These parameters defined 4 different day templates for clock controlled peak switching. Each template can define up to 10 events. Each event specifies when (hour/minute) a peak must be switched on (interfloor, up-peak, down-peak or automatic).
Each template must always have the ten lines to allow to add a peak switch with CADI if wanted. The not used lines must have a value bigger than 23 in the hour field.
If no clock switching is wanted but only automatic mode: define first event with automatic (a time must be given but when is not important, use midnight as default) and all remaining events as not used (see 71 in example below). This can also be used as default for the day templates which are not used.

Peak:
0 = automatic
1 = interfloor
2 = up-peak
3 = down-peak

Tip: to avoid to have to reset all RS print when this parameter changed simply modify the real-time clock using CADI -> this parameter will be read again.

 

Example:
 ¤5C                             *   PCT_GP_RS_PEAK
  {...                           * P_STR_TIME_ACT_DAY_TMP_1
   ¤70[06 04 01 07 1E 02]        *    7.30 : up-peak
   ¤70[06 04 02 08 00 01]        *    8.00 : interfloor
   ¤70[06 04 03 0B 1E 03]        *   11.30 : down-peak
   ¤70[06 04 04 0C 00 00]        *   12.00 : automatic

   ¤70[06 04 09 FF 00 00]        *   not used

Week Template


This table defines for each day of the week which day template has to be used. It must always have 7 lines and a template must always be specified.

If no clock control peak switching are wanted use day template 1 for each day of the week and defines each day template with first event at midnight for automatic mode and the remaining ones as not used (see P_STR_TIME_ACT_DAY_TMP_x).

day of the week: 1: Sunday, 2: Monday,… 7: Saturday
day template   : 1: P_STR_TIME_ACT_DAY_TMP_1
                 2: P_STR_TIME_ACT_DAY_TMP_2
                 …
Tip: to avoid to have to reset all RS print when this parameter changed simply modify the real-time clock using CADI -> this parameter will be read again.









Example:

 ¤5C                      *   PCT_GP_RS_PEAK
  {...                    * P_STR_TIME_ACT_WEEK_TMP
  ¤74[06 02 01 02]        * Sunday   : day template 2
  ¤74[06 02 02 01]        * Monday   : day template 1
  ¤74[06 02 03 01]        * Tuesday  : day template 1
  ¤74[06 02 04 01]        * Wednesday: day template 1
  ¤74[06 02 05 01]        * Thursday : day template 1
  ¤74[06 02 06 01]        * Friday   : day template 1
  ¤74[06 02 07 02]        * Saturday : day template 2
  ...
  }
