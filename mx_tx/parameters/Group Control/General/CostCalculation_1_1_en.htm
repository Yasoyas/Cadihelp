Cost Calculation

Summary

 This section describes the parameters in the parameter window Cost Calculation of the parameter group General in subsystem Group Control.

Max.Number of Trips

Maximum number of trip before aborting when calculating the cost of serving floor calls.

 














Bonus for Allocated


Cost substracted to the real cost for the lift that has the corresponding floor call already allocated to it. The maximum value is checked by software and corrected if bigger than P_STR_FFA_DOOR_OPEN_TIME / 2 + P_STR_FFA_DOOR_MOVING_TIME.





















Out Cost Factor


Multiplication factor for the outside costs (costs due to car position, drive and door status ,...). It allows increasing the importance of outside costs regarding inside costs. A value of 1 means same importance.

Hint: by setting this parameter to 5 (maximum value) the influence of the internal costs is very limited. This way the effect of elevators bypassing floor calls sometimes, which is typically observed on elevators with glass doors, can be reduced.





















Inter Floor Time


Time necessary in fullspeed to cover a standard floor distance (4 m). Used by the group algorithm.
Formula: (4 / speed) * 10

Example:
1   m/s -> 40 (4 s)
1.6 m/s -> 25 (2.5 s)  (4 / 1.6 * 10)
2   m/s -> 20 (2 s)
3.5 m/s -> 11 (1.1 s)
4   m/s -> 10 (1 s)





















Door Moving Time


Time necessary to open or closed a door. Used by the cost calculation. There is only one value for the whole group.





















Idle Time


Time interval when the cost calculation task is suspended. It allows other tasks with lower priority to execute. If this value is increased the cost calculation will be slower.





















Time Lost per Stop


Time lost for an intermediary stop in comparison with a direct trip. Used by the group algorithm. Only one average value for the whole group.





















Door Open Time


Door open time used by the cost calculation. There is only one value for the whole group.





















Blind Zone Extra Cost


It defines the extra cost per meter of blind zone added to the cost of a call when crossing a blind zone.
Remark: set this vlaue to zero to disable the blind zone extra cost feature.
See also parameter P_STR_BLIND_ZONE_MIN_LENGTH.





















Blind Zone min.Length


It defines the minimum length in meter between two floors necessary to be considered as a blind zone. A blind zone will affect the cost (malus) of serving a call if the car needs to cross it to serve this call. 
Remark: if this value is higher than the biggest floor distance for this commission, this extra cost feature is disabled (no blind zone => no malus).





















Floor Distances


List containing distances between consecutive floors (first value: distance between floor 1 and 2). Floor values are group related.
Used by the group algorithm.








Example: 

Disposition:                 Parameter
                         +---------+-----------+
 floor#  L1   Distance   | INDEX   | DISTANCE  |
       +---+             +=========+===========+
    4  |   |             |  1      |   10      | floor 1->2
       +---+  4          +---------+-----------+
    3  |   |             |  2      |    4      |       2->3
       +---+  4          +---------+-----------+
    2  |   |             |  3      |    4      |       3->4
       +---+             +---------+-----------+
       | X |  10         +---+ 
    1  |   | 
       +---+













Door Reopen Mode


Door reversing mode when a floor call is registered on the floor where a car is currently standing. If the doors of the car standing at the floor corresponding to the new floor call are closing, it allows to choose if the floor call will be served immediately by this car (in that case the door will reopen) or not (doors continues to close).

 1: doors are allowed to reopen (default).
 2: doors are not allowed to reopen => call allocated to another lift or not immediately served.
