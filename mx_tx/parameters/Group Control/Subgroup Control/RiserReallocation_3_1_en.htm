Riser Reallocation

Summary

 This section describes the parameters in the parameter window Riser Reallocation of the parameter group Subgroup Control in subsystem Group Control.

Riser Reallocation

The parameter table defines the riser reallocation. A reallocation specifies in which Mode which riser must be reallocated from which subgroup to which subgroup when all the lifts of the subgroup the riser belongs to are unavailable. Up to 16 reallocations can be defined.

In order to allow to add reallocation definitions using CADI this table must always be complete (16 lines, 5 data) in PCT. The not defined reallocations have 0 as data.

The subgroup number corresponds to the ones defined in parameters P_STR_SG_SUBGROUP and P_STR_SG_RISER. The riser number corresponds to the ones defined by parameter P_STR_SG_RISER.

 

Example:
  ¤5D                                  *   PCT_GP_RS_SUBGROUP
    {...
    ¤78[06 05 01 01 02 01 03] * in Mode 1, riser 1 from sg 2 to 3
    ¤78[06 05 02 01 03 02 02] * in Mode 1, riser 2 from sg 3 to 2
    ¤78[06 05 03 01 04 03 03] * in Mode 1, riser 3 from sg 4 to 3
    ¤78[06 05 04 02 05 01 03] * in Mode 2, riser 1 from sg 5 to 3
    ¤78[06 05 05 02 05 02 03] * in Mode 2, riser 2 from sg 5 to 3
    ¤78[06 05 06 02 03 03 05] * in Mode 2, riser 3 from sg 3 to 5
    ¤78[06 05 07 00 00 00 00] * not used
    ¤78[06 05 08 00 00 00 00]
    ...
    ¤78[06 05 10 00 00 00 00]
    ...
    }

Default value: no riser reallocation.

  ¤5D                                  *   PCT_GP_RS_SUBGROUP
    {...
    ¤78[06 05 01 00 00 00 00] * no reallocation
    ¤78[06 05 02 00 00 00 00]
    ...
    ¤78[06 05 0E 00 00 00 00]
    ¤78[06 05 0F 00 00 00 00]
    }
