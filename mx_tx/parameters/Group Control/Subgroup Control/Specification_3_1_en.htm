Specification

Summary

 This section describes the parameters in the parameter window Specification of the parameter group Subgroup Control in subsystem Group Control.

Lift Allocation

The parameter (table) defines which lifts belong to which subgroup. It is possible to define up to 8 different subgroups. Important: a lift may be part of more than one subgroup if necessary (e.g.: double duplex control). For a standart elevator (group) with no subgroup control the parameter has a default value corresponding to a subgroup (#1) which includes all lifts.

Together with the parameter P_STR_SG_RISER a complete definition of the subgroup(s) and the related effective risers is made. The index number (subgroup number) in this table corresponds to the index number for P_STR_SG_RISER.

In order to allow to add subgroup definitions using CADI this table must always be complete (8 lines with 9 data) in the PCT. For subgroups not used the lift numbers in the corresponding fields are set to 0.

Riser Allocation

This parameter table defines which risers (set of floor calls) are effective for which subgroups defined in P_STR_SG_SUBGROUP. Or in other words, which landing calls are served by which subgroup (a subgroup may consist of a single lift). The floor call signal (DE-U/D) includes a riser number in addition to the floor and side attribute. This riser number for the floor calls is either defined with the signal definition in the PCT for TX-GC/TX-R5 (Group LON bus) or is taken from the parameter P_IO_RISER_ID_FLOOR (Riser Id.) for MX-GC/TX-GC2.

Together with the parameter P_STR_SG_SUBGROUP a complete definition of the subgroups and the related effective risers is made. The index (subgroup number) in this table corresponds to the index number for P_STR_SG_SUBGROUP.

Remark: In order to allow to make subgroup definitions using CADI this table must always be complete (8 lines, 9 data) in PCT. For subgroups not used the riser number is 0.

Activation Mode

This parameter defines which subgroups are active in subgroup modes 0 to 7. A subgroup mode defines which subgroups are simultaneously active. A maximum of 4 subgroups can operate in a subgroup mode. Up to 8 modes (0..7) can be defined. The default value for P_STR_SG_MODE must include at least one subgroup for mode 0 because this mode is active when no subgroup switching is required. The switching between subgroup modes is done with the group input JUG; this signal includes an attribute which defines which mode (0..7) shall be active. The parameter P_STR_SG_SUBGROUP defines which elevators are part of each subgroup used in this parameter.

In order to allow to add mode definitions using CADI, this table must always be complete (8 lines, 5 data) in PCT. For all modes which are not defined the subgroup numbers are 0.

The subgroup number used here correspond to the ones defined in parameters P_STR_SG_SUBGROUP and P_STR_SG_RISER.

Riser Id.

Definition of the riser Id. for landing calls (ZS1 and ZS20) used by I/O handler. With the introduction of the plug & play fixtures and the new (TX-GC2) user bus architecture the definition of the riser Id. for the landing calls is done with this parameter. All landing calls (DE-k) connected to an elevator will have the riser Id. given with this parameter separately for front and rear side landing calls.

Attention: Because this parameter is used to configure the IO's it is necessary to make a dobule reset after the parameter is modified. Only then the floor call IO's are reconfigured.

Normally all landing calls connected to an elevator have the same riser Id. In this case the floor index value (Field1) will be 0 and the PCT has a single entry (not per floor).
example for PCT entry:
C0 [ 06 03 00 01 01 ] riser Id. 1 for all landing calls front and rear side

If some landing calls must be defined with a different riser Id. this can be achieved by adding a line in the PCT for these levels with the correct riser definition.
example for PCT entry:
C0 [ 06 03 00 01 01 ] riser Id. 1 for all landing calls except landing call floor 6 with riser Id.2
C0 [ 06 03 06 02 02 ]
