Display General

Summary

 This section describes the parameters in the parameter window Display General of the parameter group Display in subsystem Group Control.

Low Cost Position Display

Indication of car position on car operating panel (COP) with blinking car call acknowledge lamp (LC). Can only be used on elevators with a conventional car operating panel.

Modes:

0: (OFF) - Position indication with blinking LC disabled
1: Actual floor selector value (P_DISPL_LC_SELECTOR_POS) drives corresponding car call acknowledge lamp.
2: Actual floor position value (P_DISPL_LC_FLOOR_POS) drives corresponding car call acknowledge lamp.

 














Position Display


Name of the specified floor (maximum 4 characters). There is one parameter per stop/floor. The floor name can be one to four ASCII characters. The remaining characters have to be filled with 0 (zero not 'O').

For Miconic 10 only:
The floor names for a Miconic 10 installation can only contain the ‘-‘ sign and the characters corresponding to the digits 0 to 9 (this is due to the keys available on the terminals used to enter the destination floor). A floor name can only have three characters maximum.
The range for the floor names is: -99,-98, …, -2, -1, 0, 1, 2, 3, 4, …, 10, 11, …, 99, 100, …156. Typically the main floor is ‘0’. and the ‘-‘ sign is used for floors below the main floor.

Default values for Miconic 10:  0...(number_of_floors – 1).

Express Zone


Table for definition of lift related express (blind) zones. A maximum of 3 express zones can be defined.
The definition of express zones is required to control position indicators which show additional characters (defined with parameter P_DISPL_ASCII) when the lift is travelling in a express zone. The definition for each express zone includes the floor numbers below and above the express zone and the number of intermediate (virtual) floor levels between the two floors.

When the lift is travelling in a express zone where no normal floor selector or floor position messages is sent, the travel control sends additional messages (so called TUEM_SELECTOR messages) which are used to generate the OUT_DISPLAY / VXT_EXPRESS_ZONE message to control the position indicator.
The TUEM message represents the present car position and consists of a two digit number x, y
x = closest group floor position number below present car, 
y = percentage (0-9) of group floor position of present car position related to x (e.g. 3,4 means car is at a positon 40% above floor 3

Based on the value of the TUEM_SELECTOR message and the definition of the express zone parameter the required VXT_EXPRESS_ZONE messages are generated. 
The number of intermediate floor position in a express zone is limited by the resolution of the TUEM_SELECTOR messages which is 6m. Therefore express zone display only works on elevators (simulators) where the floor to floor distance is well above 6m for the express zones defined in the PCT.

Floor Name Upper Deck


Name of the floor (maximum 4 characters) served by the upper deck of a double deck elevator related to the (group) stop number of the car. There is one parameter per stop (index number). The floor name can be one to four ASCII characters. The remaining characters have to be filled with 0 (zero not 'O').
The parameter is only used for DD elevators. It is used to convert calls entered on the in-car terminal to the stop number of the car.

For Miconic 10 only:
The floor names for a Miconic 10 installation can only contain the ‘-‘ sign and the characters corresponding to the digits 0 to 9 (this is due to the keys available on the terminals used to enter the destination floor). A floor name can only have three characters maximum.
The range for the floor names is: -99,-98, …, -2, -1, 0, 1, 2, 3, 4, …, 10, 11, …, 99, 100, …156. Typically the main floor is ‘0’. and the ‘-‘ sign is used for floors below the main floor.

Default values for Miconic 10:  0...(number_of_floors – 1).
