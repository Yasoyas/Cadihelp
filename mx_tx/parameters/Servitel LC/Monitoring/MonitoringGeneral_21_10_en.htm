Monitoring General

Summary

 This section describes the parameters in the parameter window Monitoring General of the parameter group Monitoring in subsystem Servitel LC.

Break Down Limit

The 'Break Down Limit' value specify the amount of same breakdown messages sent to the control center until the system quits to send further error messages of the same type to the control center. A special message will be generated to inform the control center of blocking those failure messages if the given value is reached. This prevents blocking of a lift system (telephone line) due to a single repeating error. 














Max. Fault / Period


Max. number of calls for faults sent to the Servitel Control Center between two calls.





















Trips before Normal Traffic


The value 'Trips before Normal Traffic' specify the necessary amount of succsessfull trips until a 'Normal Traffic OK' - signal can be generated and sent to the Control Center. 





















Trips till Normal Traffic


Number of successful trips until the status + Normal Traffic is reached. Note: A successful trip contains a door opening and closing sequence and a trip from one floor to the next.





















Error Delay


After a Warning has occurred and Error Delay has elapsed Servitel-LC starts Test Trips if enabled. If they are not successful, Servitel-LC converts the Warning into an Error.





















Time until Warning when Power Failure


The 'Warning when Power Failure' value specify the time during a powerloss has to occur until an error can be generated and sent to the Control Center. 





















First Fault Timer


The 'First Fault Timer' interacts with all controller types except SMART. It defines the time delay after receiveing a warning message until the warning turns into an error.
Before turning into an error there is a possibility to launch as much testtrip commands to the elevator controller as needed to produce a 'Normal Traffic OK' - signal.
If a 'Normal Traffic OK' - signal is received before the timer stops (Testtrip disabled) or the signal is thrown by successful test trips the warning message will be droped into the DataBuffer. 





















JAB


In service and out-of-service.





















BR


Fire service on and off.





















Test Trip Enable


The Test Trip Enable value must be switched ON to allow testtrip commands to the elevator controller to produce a 'Normal Traffic OK' - signal.





















Test Trip Delay


The value Test Trip Delay specify the time delay between two testtrip commands automatically sent to the elevator controller in the event of a controller detected emergency.





















JRV


Revision on and off.





















SRE


Reservation on and off.





















Wire Breakage


The Wire Breakage value holds the timelimit to detect a wire breakage on the DAKI or DAKA inputs. If one of the buttons has a pressed signal longer than the time specified an error will be generated and sent to the Control Center.





















Test DA-Input


Telemonitoring parameter





















NS


Standby supply on and off.





















Normal Traffic


Normal traffic on after Analysis was off.





















Service Visit


Service Visit on and off.





















Send State with Error


Send elevators status when an Error occurs.





















Warning to Local Buffer


Store Warning in a separate Buffer. Use this feature when special services like Reservation are often switched on and off.





















Test Trips


Enable/disable test trips. If enabled, value of parameter Trips till Normal Traffic will be activated.





















Test Trip Period


Time until a new attempt to perform a test trip is made after the last test trip (attempt).
