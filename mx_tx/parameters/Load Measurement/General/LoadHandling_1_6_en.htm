Load Handling

Summary

 This section describes the parameters in the parameter window Load Handling of the parameter group General in subsystem Load Measurement.

Nominal Load

Nominal car load [kg]. For double deck elevators this is the sum of the nominal loads for the lower and the upper deck. E.g. if a DD elevator is specified with a nominal load of 2 x 1600kg then the value for P_ANT_NOM_LOAD is 3200kg.

On first generation VX/TX systems, nom_load was defined twice (also in the LIFT subgroup of the PCT). For TX-GC2/MX only the P_ANT_NOM_LOAD is required.

The calculation of the drive internal load limit values for the load-measuring system is based on the parameters NOM_LOAD and CODE_TYPE
for NA:
kl_h_load = NOM_LOAD / 2 ,
kl_max_load = NOM_LOAD * 1.1 ,
kl_x_load = NOM_LOAD * 1.3 ,

others (EUAP):
kl_h_load = NOM_LOAD / 2 ,
kl_x_load = NOM_LOAD * 1.1 ,

these limits are independent of load limits defined in PCT_GP_RS_GENERAL

 














Auto-Adjust Delay


Enable/disable autoadjust for zero-load with load measuring: HP-LMS.

0: OFF
1..255: auto adjust ON with a period of x days






















Minimal Load Limit


Value specifying the minimal load limit in [kg]. If the measured car load is below this limit the minimal load function is activated. A value of zero implies that this minimal load function function is disabled.
With load contatcs (see parameter LM_TYPE) the minimal load value is used to calculate the load percentage when the input KL-M is active.





















Full Load Limit


Value specifying the full load limit in percent (%) of the nominal load. When the car load exceeds the limit the full load function () is activated (floor calls no more served). -> Same effect as with activation of input KL-V.





















Over Load Limit


Value specifying the overload limit in percent (%) of nominal load. When car load exceeds the limit the over load function () is activated. -> Same effect as with activation of input KL-X.





















Load per Person


Defines the average weight of one person in kilograms (kg). Used to convert nominal cabin load in kg to the maximum number of passengers.





















Minimal Load Control


Configuration for minimal load control (Anti Nuisance). Cancel excessive car calls in normal operation (only required for KS and KA control mode).

Parameter values:
0: minimal load control off.
1: minimal load control with load measurement. Works only with DMS or KL-M.
2: minimal load control with door photocell PHT. This mode is required
   if minimal load control is ordered, but no DMS and no KL-M are
   configured.

Load always N Service
IF LOC_ANTI_NUISANCE is ordered
    IF any LOC_DMS is ordered 
    THEN
        SET P_STR_MINIMAL_LOAD_CONTROL = 0
          (Reason: The DMS activates automaticaly the load controlled
           anti-nuisance, which is more sophisticated than the minimal
           load control function with parameter = 1)
    ELSE    
        SET P_STR_MINIMAL_LOAD_CONTROL = 2
    END IF
ELSE
    IF any LOC_DMS is ordered
    THEN    
        SET P_STR_MINIMAL_LOAD_CONTROL = 0 (default value)
    ELSE
        SET P_STR_MINIMAL_LOAD_CONTROL = 0 (default value)
    END IF
END IF





















Anti Nuisance


Limit for number of registered car calls based on number of passengers in the car.

Value 1..10: Allowed difference between the number of passengers in the cabin and the maximum number of car calls. If there are more calls than allowed, the surplus car calls will be canceled.
Value 0 : Anti nuisance function disabled. (Anti nuisance is also disabled if no DMS is configured.)





















LX Signalization Mode


Parameter to control the behaviour of the overlad signals (lamp and buzzer) on the COP (car operating panel). The signals controlled by the parameters are: LL-X, SUMC, SUMLX

behaviour of the outputs when overload is active (depending on the parameter value):
< 2:  LL-X blinking, SUMC on, SUMLX = off
> 6:  LL-X blinking, SUMC on, SUMLX = off
2:  LL-X blinking, SUMC off, SUMLX = on
3:  LL-X blinking, SUMC on, SUMLX = on
4:  LL-X on, SUMC on, SUMLX = off
5:  LL-X on, SUMC off, SUMLX = on
6:  LL-X on, SUMC on, SUMLX = on
