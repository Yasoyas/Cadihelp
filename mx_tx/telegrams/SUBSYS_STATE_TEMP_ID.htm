Specification
Telegram Group: SUBSYS_STATE
Telegram: TEMP_ID
Attribute 1: T_SUBSYS_ATTR_1
Attribute 2: -
Datalength: 0

T_SUBSYS_ATTR_1

 1:
This telegram contains the NACK, RECONFIGURATION and NOTOKEN counters on the liftbus from a subsystem

3:
attr_1 : VXT_IOBUS_DIAGN
attr_2 : --
data_length: 6
data[0] : control register of IOBUS controller
data[1] : status register of IOBUS controller
data[2] : IOBUS-block which is currently shifted
data[3] : current IOBUS-timer value
data[4] : IOBUS interrupt counter (high byte)
data[5] : IOBUS interrupt counter (low byte)
Description: This telegram contains internal states of the IOBUS
hard- and software

4:
attr_1 : VXT_LIFTATTENDANT_STATE
attr_2 : --
data_length: 0
Description: A VCOM_get with this telegram forces the RC-SW to reply
all the liftattendant EVENT_DESCRIPION telegrams.

7:
DATA[0] == TRUE Doorbridging device (LON node like LONIC, LONGIC) is available
DATA[0] == FALSE Doorbridging device (LON node like LONIC, LONGIC) is not available
DATA[1] == TRUE communication to doorbridging device lost
DATA[1] == FALSE communication to doorbridging node ok
