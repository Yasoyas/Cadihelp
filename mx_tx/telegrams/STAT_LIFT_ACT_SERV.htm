Description: defines the current service with the highest
priority and the all active services

cur_service_id: contains the current active running
service_identifier.
next_phase : contains the next phase number of the current service.
cur_serv_prio: contains the priority of the current service


data[1..13]: Statuses of the services 1..100, active = 1, inactive = 0.
Service_id 1 corresponds to Bit 0 of data Byte 1
Service_id 2 corresponds to Bit 1 of data Byte 1
Service_id 100 corresponds to Bit 3 of data Byte 13
Service_id values defined in VXT_T_SERV_MODE.

In the following example are service 3 and service 11 activ.
