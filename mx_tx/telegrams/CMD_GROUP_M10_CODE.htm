CMD_GROUP (4)  /  M10_CODE (13)

Description

 attr_1 : vxt_M10_terminal_type
attr_2 : --
data_length: 9..11
data[0] : boarding floor or car number
data[1] : boarding side
data[2] : terminal position
data[3] : node_type ( high byte)
data[4] : node_type ( low byte)
data[5] : node_id (high byte)
data[6] : node_id (low byte)
data[7] : ASCII code 1st byte

data[8] : ASCII code 2nd byte
data[9] : ASCII code 3rd byte
data[10] : ASCII code 4th byte

Description: M10 code.
The data [0] and [1] are boarding floor and side if attr_1 is
floor or virtual terminal. If attr_1 is car terminal only data[0]
is used to hold car number.

M10 terminal type:
1: floor terminal
2: car terminal
3: CADI terminal
4: Lobby Vision terminal

terminal position: 0..23, position of the terminal that was used to
enter the code.

The node_type and node_id are used to address the terminal (not necessary
for virtual terminal).
The length of the code is variable (2..4). The maximum value for the code
is 8191.

T_VALUE

 This Type is used for general data description.
See description above for more details.
range: 0..255

T_TERMINAL_POSITION

 1..max, position of the terminal where the M10 call is given.

The node_type and node_id are addresses used to identify the terminal where the M10 call has been generated.
