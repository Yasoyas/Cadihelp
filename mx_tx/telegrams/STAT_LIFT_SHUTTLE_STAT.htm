STAT_LIFT (7)  /  SHUTTLE_STAT (24)

Description

 attr_1 : state type
attr_2 : state id
data_length: variable, depends on attr_1 and attr_2

Description: Lift state for shuttle control.

attr_1 : 1, acknowledge of command from the group
attr_2 : command id as in telegram VXT_SHUTTLE_CMD attr_1.
data_length: 1
data[0] : 0: not OK (command cannot be executed)
1: OK (command was or will be executed)

attr_1 : 2, current shuttle state machine state
attr_2 : --
data_length: 1
data[0] : state as defined in stsw_def.h / SM_SHUTTLE_STATE_TYPE
0: unknown
1: travelling
2: waiting at floor
3: additional load car
4: primary load car
5: unavailable for shuttle

attr_1 : 3, primary load car leaves the floor

attr_2 : --
data_length: 1
data[0] : floor number (that primary load car left).

attr_1 : 4, event primary load car expired. Only for Tx4.
attr_2 : --
data_length: 1
data[0] : floor number (where car is standing).

attr_1 : 5, event new car call at other shuttle floor
for additional load car.
attr_2 : --
data_length: 0

attr_1 : 6, event car is standing at shuttle floor.

attr_2 : --
data_length: 1
data[0] : floor number where car is standing.
