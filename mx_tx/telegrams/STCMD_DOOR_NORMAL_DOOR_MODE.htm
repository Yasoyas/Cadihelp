Description: door_mode for normal_operations

This telegram is used to define the priority and behaviour of each of
the defined reversing devices. The behaviour of a reversing device is
defined in one byte as depicted in the following picture. The definition

order determines the priority of the corresponding reversing device.
This means that reversing device KSKB has the highest priority and DTS
the lowest. Byte 6 can also contain a reversing device definition but
there is not one defined at the moment.

1) Reversing device ID: (4 Bits)
As defined in vx_tel_def.h (VXT_T_DOOR_REVERSING_DEVICES).
The order specifies the priority of the reversing devices.

2) Operating Mode: (3 Bits)
The behaviour of the reversing device is defined for the
positive and/or negative edge.

------------------------------------------------------
| Operating Mode | Positive Edge | Negative Edge |
------------------------------------------------------
| 1 | reversing | ............... |
| 2 | reversing | stop |
| 3 | reversing | reversing in the|
| | | other direction |

| 4 | Stop | ............... |
| 5 | TOHZ = 0 | ............... |
| 6 |Stop immediately,| ............... |
| |when longer | ............... |
| |active reversing | ............... |
| 7 | Reserved |
------------------------------------------------------

3) activ/inactiv: (1 Bit)
A reversing device is allowed (activ) when the value == 1
or blocked (inactiv) when the value == 0.

4) Opentime reduction Mode:
1: Normal door opened time (given in cmd door open)
2: Reduced (until minimal) door opened time (50% each time)
3: Minimal door opened time
