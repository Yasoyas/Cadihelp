CMD_GROUP (4)  /  GROUP_CALL_LOCK (5)

Description

 attr_1 : --
attr_2 : VXT_T_CALL_LOCK_SOURCE
data_length: 4
data[0] : call_type_1
data[1] : first locked floor
data[2] : last locked floor
data[3] : vxt_state

Description:
Command to lock cabin- or/and floorcalls on the specified lift,
side and floor.
Simulated calls are not supported.

attr_2: VXT_T_CALL_LOCK_SOURCE, source that locks or
unlocks the calls.

data[0]: call_type_1, specifies call type and side.


128 64 32 16 8 4 2 1 <-- value(10)
+---+---+---+---+---+---+---+---+
| 1 | 1 | 0 | 1 | 1 | 1 | 0 | 1 | call_type_1: BYTE-MASK
+---+---+---+---+---+---+---+---+
7 6 5 4 3 2 1 0 <--BIT_NR

In this example with the value 221 are locked all cabin- and

floor_calls on side_1 and side_20.


BIT_NR CALL_TYPES
0 Cabin_Call from VXT_SIDE_1
1 Simulated Call from VXT_SIDE_1
2 Up_Floor_Call from VXT_SIDE_1
3 Down_Floor_Call from VXT_SIDE_1

4 Cabin_Call from VXT_SIDE_20
5 Simulated Call from VXT_SIDE_20
6 Up_Floor_Call from VXT_SIDE_20
7 Down_Floor_Call from VXT_SIDE_20

data[1]: First floor to lock; value 0 means all floors

data[2]: Last floor to lock; value 0 means all floors

data[3]: VXT_STATE_1 means locked, VXT_STATE_0 means unlocked.

Specification
Telegram Group: CMD_GROUP
Telegram: GROUP_CALL_LOCK
Attribute 1: -
Attribute 2: T_CALL_LOCK_SOURCE
Datalength: 4
Data[0]: T_VALUE
Data[1]: T_VALUE
Data[2]: T_VALUE
Data[3]: T_STATE

T_CALL_LOCK_SOURCE

 1: VXT_ALL_LOCK = 0 /* all sources HLS_1 VXT_LOBBY */
2: VXT_HLS_1_LOCK /* task HLS on RS print */
3: VXT_HLS_2_LOCK /* task HLS on RS print */
4: VXT_HLC_LOCK /* task HLC on RS print */
5: VXT_HCB_LOCK /* task HCB on RS print */
6: VXT_KEY_LOCK /* Key as hardware input */
7: VXT_HW_INP_LOCK /* Hardware input of any type */
8: VXT_LOBBY_LOCK /* LOBBY as call lock source */
Value Name
0 ALL_LOCK
1 HLS_1_LOCK
2 HLS_2_LOCK
3 HLC_LOCK
4 HCB_LOCK
5 KEY_LOCK
6 HW_INP_LOCK
7 LOBBY_LOCK

T_VALUE

 This Type is used for general data description.

range: 0..255
