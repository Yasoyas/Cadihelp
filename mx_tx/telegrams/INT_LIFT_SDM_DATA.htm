INT_LIFT (23)  /  SDM_DATA (19)

Description

 tel_grp_id : VXT_INT_LIFT
tel_id : VXT_SDM_DATA
attr_1 : vxt_sdm_query
attr_2 : varying according to attr_1
data_length: varying


attr_1 : VXT_SDM_PRESENT
attr_2 : --
data_length: 1
data[0] : vxt_state

Description : The pysical presence of the ChipCard is returned
by a VCOM_get() on the above info. Any application
may perform the get(). The result is returned only

by the ChipCard handler.
VXT_STATE_1 = ChipCard is present
VXT_STATE_0 = ChipCard Socket is empty or malfunctioning

attr_1 : VXT_SDM_VALID
attr_2 : --
data_length: 1
data[0] : vxt_state

Description : The ChipCard is validated, and the result returned by a
VCOM_get() on the above info. Any application may perform
the get(). The result is returned only by the ChipCard

handler.
VXT_STATE_1 = ChipCard is valid for this lift, bank and
installation number
VXT_STATE_0 = ChipCard is invalid or missing


attr_1 : VXT_SDM_COMMAND
attr_2 : vxt_sdm_commands
data_length: varying according to attr_2

attr_2 : VXT_SDM_INVALIDATE
data_length: 1
data[0] : VXT_STATE_0

Description : Command from SMLCD or CADI to the ChipCard handler.

The current ChipCard authentification key is erased
and the currently installed ChipCard is no longer usable.
VXT_STATE_0 is required or the command is ignored.
Subsequent VCOM_get() for VXT_SDM_VALID retruns VXT_STATE_0
until a new ChipCard is installed and validated.

Specification
Telegram Group: INT_LIFT
Telegram: SDM_DATA
Attribute 1: -
Attribute 2: -
Datalength: 1
Data[0]: T_STATE

T_STATE

 State ON, OFF and Blinking:
possible states of outputs or inputs

and

Priority Ready and Fully Ready:
possible states of bus (LON1, LON2 and BIO) and
STATUS IO-Handler Telegrams
Value Name
0 STATE_0
1 STATE_1
2 STATE_BLINKING
3 PRIORITY_READY
4 FULLY_READY
241 WILDCARD
255 UNDEFINED
