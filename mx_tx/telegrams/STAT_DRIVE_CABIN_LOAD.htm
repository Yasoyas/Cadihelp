Description

 attr_1 : --

For single deck:
data_length: 2
data[0] : high_byte of cabin load
data[1] : low_byte of cabin load
For double decks:
data_length: 4
data[0] : high_byte of cabin load of lower deck
data[1] : low_byte of cabin load of lower deck
data[2] : high_byte of cabin load of upper deck
data[3] : low_byte of cabin load of upper deck

Description: Percentage load from the cabin_nominal_load

cabin_load: The cabin load is in per-mille proportional to the
nominal load of the cabin. In the case where load
contacts are present, the corresponding value in
per-mille must be generated.

These values will be retrieved from the RS-PCT.

The values for full- and overload are stored in

percent and need to be converted into per-mille
before being sent out. The minimal load value is
stored in kilogrammes and must be converted into
per-mille by dividing the minimal load value by
the nominal load value which is also in kilogrammes.
