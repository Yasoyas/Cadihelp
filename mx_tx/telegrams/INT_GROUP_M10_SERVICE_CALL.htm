INT_GROUP (24)  /  M10_SERVICE_CALL (13)

Description

 attr_1 : vxt_M10_terminal_type
attr_2 : vxt_M10_call_type
data_length: 10
data[0] : boarding floor number
data[1] : boarding floor side
data[2] : exiting floor number
data[3] : exiting floor side
data[4] : number of person
data[5] : terminal position
data[6] : node_type (high byte)

data[7] : node_type (low byte)
data[8] : node_id (high byte)
data[9] : node_id (low byte)

Description: M10 call for a service (call given after having typed
the service code for service requiring a call).
The exiting side data is not used currently (always all sides)
but is foreseen for later when M10 will handle two separate doors
also for exiting.

M10 terminal type:
1: floor terminal

2: car terminal
3: CADI terminal
4: Lobby Vision terminal

M10 call type:
1: normal call
2: handicapped call

terminal position:
1..max, position of the terminal where the M10 call is given.

The node_type and node_id are addresses used to identify the terminal
where the M10 call has been generated.

The floor/side of a call generated from a car terminal are contained in

data[2] and [3]. Data[0] and [1] are undefined.
