Pulse Width

Summary

Some output telegrams (corresponding to a lamp generally) have a special data called pulsewidth. This special data can be used the following way:

a) vxt_state is VXT_STATE_0: pulsewidth not relevant

b) vxt_state is VXT_STATE_1:
    pulsewidth > 0: defines for how many 100ms ticks the output must be active.
    pulsewidth = 0: the output stays constantly on until a telegram with VXT_STATE_0
    occurs.

Example:
vxt_state = VXT_STATE_1

pulsewidth = 10 (10*100ms -> 1s)

-> Output signal:


c) vxt_state is VXT_STATE_BLINKING:
    pulsewidth > 0: defines the time the output is on and off when blinking.
    pulsewidth = 0: a default time (0.5s) is used.

Example:
vxt_state = VXT_STATE_BLINKING
pulsewidth = 2 (2*100ms -> 200ms = 0.2s)
