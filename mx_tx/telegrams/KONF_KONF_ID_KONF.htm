Description: Give data from the PCT

Dient fuer die Anfrage, als auch fuer Verteilung der
PCT-Version und fuer die GET- und REPLAY-Telegramme
bei der physikalischen Konfiguration. Die genaue

Beschreibung des Telegramms ist in Dokument 'EZ-ST
90 147' und 'EZ-ST 90 223'.

konf_id: Auf dem ersten Attribut wird beim Konfigurations-
(0..511) Telegramm definiert, welche Parametergruppe oder
von welchem Subsystem die Input/Output mit den im
Telegramm enthaltnen Daten konfiguriert werden.
Falls der data_id den Wert PCT_INPUT_OUTPUT enthaelt,
definiert 'konf_id' zu welchem Subsystem die nachfol-

genden Input/Output-Konfigurationsdaten gehoeren oder
von welchem sie angefordert werden.

Falls der data_id den Wert '3' enthaelt, ist der konf_id = FFh

und in data[0] ist die PCT_Versions- Nummer enthalten oder wird
gefordert.

konf_state_id:
Kennzeichnet das erste und das letzte VXT_KONF-
Telegramm mit den folgenden Werten:
9 PCT_KONFIG_START
10 PCT_KONFIG_END

11 PCT_KONFIG_ERROR

data_id: Steht auf dem 2. Attribut vom Konfigurationstele-
(0..15) gramm und definiert die folg. Konfigurationsarten:
1 PCT_INPUT_OUTPUT Input/Output-Konfiguration
2 PCT_GROUP_PARAMETER Konfiguration von Parametern

konf_data: In diesen Daten sind die mit den Attributen ge-
kennzeichneten Konfigurationsdaten enthalten.

Falls es Konfiguratinsdaten fuer einen Input/Out-

put sind, haben die konf_data folgende Struktur:
data[0]: Nummer vom Stecker, an dem das Element
nach Pinbelegungsplan angeschlossen ist.
data[1]: Nummer von jenem Pin,an den das Element
nach Pinbelegungsplan angeschlossen ist.
data[2]: Definiert IO_Steuerart folgendermassen:

7 0 Bit_Nummer

---------------------------------
| 1 | 0 | 0 | 1 | 1 | 1 | 0 | 1 | IO_STEUERART
--^---^---^---^---^---^---^---^--
| | | | | | | |
| | | | | | ------- ACTIVE
| | | | --------------- QUIT_ART
| | ----------------------- FLANKE
------------------------------- IN_OUT

In_Out: Vom Print des jeweiligen VX-Subsystems
aus gesehen:
0 Nicht definiert
1 Output z.B. Lampen
2 Input z.B. Schalter
3 Inp/Out Pin dient fuer Input
und Output (z.B. DE)

Flanke: Definition der Ansprechflanke (nur bei

Inputs):
0 Nicht definiert
1 Ansprechen auf steigende Flanke
2 Ansprechen auf fallende Flanke
3 Ansprechen auf steigende und fal-
lende Flanke

Quit_Art: Definiert Art der Quittung (nur bei
Input und Output Elementen z.B. DE):

0 Nicht definiert
1 Keine Quittung
2 Autom. Quittung (bleibt auf de-
finierten Level)
3 Automatisch,zeitlich begrenzte
Quittung (Bleibt auf definier-
tem Level,wenn innerhalb einer
definiertem Zeit eine Quittung

folgt).
Active: Definiert den aktiven Level.
1 Aktiver Level bei logisch 'HI'
0 Aktiver Level bei logisch 'LO'

data[3]..data[n-1]: enthalten Value-Bytes, die
die Telegramme und die speziellen Daten
definierten. Die Telegramme sind dabei
jene, die das Subsystem beim Ansprechen

eines Inputs sendet und/oder einen eige-
nen Output aktivieren.

Falls es Konfiguratinsdaten von einer Parameter-
Gruppe sind,haben die konf_data folgende Struktur:

data[0]: Enthaelt den jeweiligen Parameter_ID

data[1]: Definiert den Datentyp vom jeweiligen
Parameter. Es sind dabei folgende Daten-

typen definiert (weitere in PCT_DEF.H) :

-----------------------------------------------------
| DATENTYP | ID | Anz-Byte | WERTBEREICH |
-----------------+----+----------+-------------------
| PCT_DT_BYTE | 1 | 1 | 0..255 |
| PCT_DT_SHORT | 2 | 2 | 0..65535 |
| PCT_DT_LONGINT | 3 | 4 | 0..(2^32-1) |

| PCT_DT_STRING | 4 | 1 | ASCII-Charakter |
| PCT_DT_REAL | 5 | 4 | IEEE-SP-REAL-TYP |
: : : : :
-----------------------------------------------------

data[2]: Definiert die Anzahl der folgenden Daten-
Bytes vom jeweiligen Parameter
data[3]..data[n-1]: enthalten die einzelnen Werte

vom jeweiligen Parameter.
