Description: command to lock car calls at the specified floor
and side for the specified lift as a temporary
locking overlay to internal floor locking
characteristic.

time-out/locking state:
0: OFF/CLEAR, deactivation of the temporary lock mask
and clearing of the timeout.

1: ON, activation of the temporary lock mask and timeout.

time-out duration: Only relevant if time-out/locking state is ON.
1..255: specify how long the temporary lock mask should
be active if no car call occurs.
0 : no time-out (infinite duration).

Floor lock mask: bit at 0 -> no lock at this floor.
bit at 1 -> lock at this floor.
Only relevant if time-out/locking state is ON.

The lowest floor per data corresponds to the lowest bit
(righest one):
bit: 8 7 6 5 4 3 2 1
floor: x+7 x+6 x+5 x+4 x+3 x+2 x+1 x

Example: floors 10, 27 and 28 with a lock, all the others
without lock:
data[1] = 0 00000000 floors 8 <- 1
data[2] = 2 00000010 floors 16 <- 9

data[3] = 0 00000000 floors 24 <- 17
data[4] = 12 00001100 floors 32 <- 25

Not all 17 data need to be used each time, the data with zero
at the end of the telegram can be omitted.
