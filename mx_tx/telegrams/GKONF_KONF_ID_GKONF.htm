Description: Get configuration

Telegramm fuer die Nachfrage eines Drivers (DKONF oder DCADI)
nach kommissionsabhaengigen Parametern. DATA_CONFIG wird
dabei ein Provide fuer die eigene PCT, als auch fuer die

jeweiligen Lift-,System- und STR-Parametergruppen machen. Die
Subsysteme werden nur fuer die eigene Parametergruppe ein
Provide ge- ben, so zum Beispiel der Antrieb fuer die
ANT-Para- metergruppe.
Wenn von einem Driver ein GET-Telegramm 'Tel_GKONF' gesendet
wird, kann die gewuenschte Parametergruppe nur dann ueber den
DCADI geaendert werden, wenn ein Login beim lokalen Driver
gemacht wurde, da ja nur dort die entsprechenden

Restriktionen definiert sind.
Eine genaue Beschreibung von diesem Telegramm ist im Dokument
'EZ-ST 90 223'.

konf_id und konf_data sind im letzten Telegramm VXT_KONF
beschrieben. Von einem GKONF-Telegramm werden nur die
Parameterdefinitionen verwendet.
