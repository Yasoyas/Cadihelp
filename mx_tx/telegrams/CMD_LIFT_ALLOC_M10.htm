data[8] : 'over the hill' flag

Description: Allocation of an M10 call to a lift (1..8).
The halfround where the call is allocated is given in absolute
value (0..4) and relative to the current halfround (1..5, where
1 is for the current halfround, 2 for the next halfround,...).
The exiting side data is not used currently (always all sides)
but is foreseen for later when M10 will handle two separate doors

also for exiting.
The 'over the hill' flag is 1 if the call to which this allocation
corresponds causes an 'over the hill' trip (for itself or for
another call), else 0.

Special case: number of passenger is 0 (data[6]). When the number of
passenger is 0 it means that a boarding had been moved
due to over the hill feature. This telegram with number

of passenger 0 is sent for every passenger that have to
board sooner due to over the hill to indicate the new
boarding halfround.
It is used to correct the Supervisory display and the
statitistics. See also telegram VXT_M10_END_BOARDING.

vxt_M10_call_type:
1: normal call
2: handicapped call

T_TERMINAL_POSITION

 1..max, position of the terminal where the M10 call is given.

The node_type and node_id are addresses used to identify the terminal where the M10 call has been generated.
