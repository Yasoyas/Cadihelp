Description

 attr_1: VXT_IO_TRACE_FIRST_TEL [1]
VXT_IO_TRACE_ONE_TEL [3] If the size of the data requires
more than one telegram, then the
id is VXT_IO_TRACE_FIRST_TEL

attr_2: VXT_IO_TRACE_ONE_BLOCK [6]
VXT_IO_TRACE_NO_BLOCK [7] There is only one block with the
id VXT_IO_TRACE_ONE_BLOCK

If no download is in progress, NO_BLOCK is sent back.

data_length: 8 + length of UI label (max. 17)

data[0] VXT_DI_DOWNLOAD_START [8] (VXT_T_CHANNEL_UNIT_STATE)
data[1] 1..255 internal board type
data[2] 0..255 unit type (0 = not used, undefined)
data[3] 0x00..0xFF unit ID (high byte)
data[4] 0x01..0xFF unit ID (low byte)
data[5] 0x01..0xFF Subnet (LON) / NetWorkAdr_MSB (BIO2)
data[6] 0x01..0x7F Node (LON) / NetWorkAdr_LSB (BIO2)

data[7] VXT_DI_NODE_STATE_DEAD [0]
VXT_DI_NODE_STATE_ALIVE [1] Connection state

data[8] DI_UNFROZEN [0]
DI_FROZEN [1] freeze state

data[9] VXT_T_CHANNEL_ID channel Id

data[10] 1. Unit name Char This user interface label represents
the unit name. The length of this name
can be up to 25 characters (2 telegrams).
data[11] 2. Unit name Char
.. ..
data[16] 7. Unit name Char
