Misc.Parameters

Summary

 This section describes the parameters in the parameter window Misc.Parameters of the parameter group Miscellaneous in subsystem Specials.

Door Location Deck

Location of Door Drive in Car Normally the location of the door drive in the car (1. side(Zugang 1) or 2. side (Zugang 2)) is defined by a jumper on the lift-control-interface PCB (e.g. the VLON70 or the LONDIM). This allows to specify either Access side 1 or Access side 2. No adjustments have to be made for this case to the door drive, because reading that jumper and forwarding that information is the default.

For some special configurations (double decker car, three accesses to car) this is not sufficient. For these cases one has to set the Location parameter to one of the values defined below and the door will provide that information to the lift control during configuration. This feature works with a serial/field bus interface only,
but not with a parallel interface.

0: LON_DOOR_NO_DECK, Jumper info is relevant only
1: LON_DOOR_DECK_1
2: LON_DOOR_DECK_2

This parameter is a setting parameter.

Varidor


Varidor contains the brand number of the door, e.g. 70 for a Varidor 70 or 50 for a Varidor 50 .
It is a setting parameter, that initiates default parameter settings. The values 700-710 are reserved for development only.

This parameter is a setting parameter.





















HT


Nominal door height in m





















Cladding_Raw


Cladding thickness
This parameter value used for calculation the door mass. The calculation is made by Door Controller itself. 
Value 0: No Cladding
Value 1: Cladding 1.5 mm
Value 2: Cladding 1.5..4 mm
Value 3: Cladding 4..8 mm
Value 4: Cladding 8..12.5 mm





















ReCalib After N Cycles


Now the parameter is used to change the Maximum drift for which the calibration is to be applied





















Locking Zone


Locking_zone is the position where locking may start during closing, i.e. where the clutch may start to release the lending door lock during nudging speed.





















Drive Delay


Delay time of the closed loop force control of the inverter in s.
The dynamics of the force control loop of the drive unit is modelled as a first order delay.
If a a voltage step at the drive input, i.e. the analogue input, is applied then the generated torque/force
will change. The time delay between the step at the input and the instant when the output value reaches 63%
of the end value is the DriveDelay.





















BKE_Raw


Nominal door width as input from ESI





















Control Amplification


Dynamic control amplification setting
Important: Lower limit > 0 (not zero!!) and Upper limit < 2*DriveDelay/0.006F
( T1 = 0.01s: 0.006F = 0.0045s -> Upper limit < 4.4, 0.006F = 0.008s -> Upper limit < 2.5 )
