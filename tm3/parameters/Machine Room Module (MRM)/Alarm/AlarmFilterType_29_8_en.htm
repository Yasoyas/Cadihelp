Alarm Filter Type

Summary

 This section describes the parameters in the parameter window Alarm Filter Type of the parameter group Alarm in subsystem Machine Room Module (MRM).

Filter Type

The Alarm Filter is used to provide correct usage of the alarm buttons. TM3 differs between two different filter types - "1-Puls Filter" and "2-Puls Filter". The filter types are described lower in this document.
NOTE: Each filter type requires several parameters for configuration. If you change one parameter value (for example T0 on 2 Puls Filter), this value will be also changed in all other Filter Types. The table at the end of this parameter description shows which parameter is used for which filter type.

1 Puls Filter

To be registered as an alarm, the alarm key (DAKI) must be pressed as long as indicated by parameter T0 (in this example: 200 x 10ms = 2s).
The voice announcement is played immediately after the alarm registration.
The call procedure starts immediately after the alarm registration.
The phone numbers which will be dialed are defined in the Alarm Sequence [see chapter 3.3.3 "Parameter - Dial Sequences"]


2 Puls Filter

The alarm key (DAKI) must be pressed two times or as long as stated in the T3 value to register and transmit the alarm to the Control Center.
The key must be pressed as long as indicated by parameter T0.
As soon as the key is released, T1 is been actuated. T1 is a delay time period. If the alarm button is pressed before T1 has expired, the second pressing of the key is not registered.
The second time the key must be pressed at least as long as indicated by parameter T2.
T3 specifies the time period in which the two described pulses have to take place. If the alarm is not recognized within this time, the alarm registration sequence is reset to the first puls detection.
In case the door opens within this sequence, the alarm registration sequence is reset to the first puls detection.
The voice announcement is played immediately after the alarm registration.
The call procedure starts immediately after the alarm recognition.
The phone numbers which will be dialed are defined in the Alarm Sequence [see chapter 3.3.3 "Parameter - Dial Sequences"]


AMD Filter

To be registered as an alarm the alarm key (DAKI) must be pressed as long as indicated by parameter T0.
The AMD-Filer inserts a delay between the registration of an alarm and making a call.
If the state of the discriminator input changes within this time, the alarm will be canceled.
The delay can be set to the following values:
Discriminator input disabled (default), 30s, 45s, 60s, 75s, 90s, 105s, 120s.
The call procedure starts when the discriminator input does NOT change its state whithin the specified discriminator time.
The voice announcement is played when the Call Sequence starts.
The phone numbers which will be dialed are defined in the Alarm Sequence [see chapter 3.3.3 "Parameter - Dial Sequences"]
If the discriminator input is used, the delay must be set to a value greater than the maximum trip time.



Adaptive Filter

Alarm registration occurs both via a single pulse and a double pulse filter.
The general idea is to have a 2-Pulse Filter in case of normal operation and a 1-Pulse Filter when the system detects a problem at the installation.

The Adaptive filter can NOT be used in TAM-Mode, because in TAM-mode the system dosn't have any monitoring information at all.

The 1-Pulse Filter is active in case of the following situations:

Loss of power supply in the control system.
Car stands still with brakes closed outside of the door area.
Fault detected.
In normal conditions during critical situations (rev. Break- / Door Delay)
The 2-Pulse Filter is active in case of all other situations (normal operation)

The 2-Puls Filter is reset to first puls detection every time when the door or drive state (KETS / KB) changes or when T3 expires.
